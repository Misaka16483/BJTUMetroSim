## 1. 调研结论摘要

本次调研认为，第 2 模块和第 3 模块应作为轨道交通仿真系统中的“运动与能量核心”。车辆动力学仿真模块负责把牵引、制动、坡度、阻力、限速等输入转化为列车的位置、速度、加速度和停车状态；牵引供电仿真模块负责模拟供电区段、电压、电流、功率限制、能耗和再生制动对车辆牵引能力的影响。

在实训项目周期较短、真实设备接口尚不完全明确的情况下，建议采用“分层实现”：

1. 最小可行版本：单列车、一维运动、离散牵引/制动级位、简化阻力、固定供电电压、能耗估算。
2. 可演示版本：加入坡度影响、制动响应延迟、供电限功率、欠压/断电故障、再生制动能量估算。
3. 增强版本：加入牵引特性曲线、制动曲线、供电区段切换、多车再生能量利用、电压跌落和功率流简化计算。

两模块之间的核心关系是：

```text
供电模块输出：供电可用性、电压、功率上限、牵引限制比例
        ↓
车辆模块使用：限制牵引力/牵引级位/最大可用功率
        ↓
车辆模块输出：速度、位置、牵引功率、制动功率、再生制动功率
        ↓
供电模块记录：能耗、再生能量、区段负载、故障影响
```

## 4. 第2模块：车辆动力学仿真模块

## 4.1 模块定位

车辆动力学仿真模块用于模拟列车在轨道上的一维纵向运动，是自动停车、人工驾驶、信号限速、供电能耗和运行评估的基础模块。

该模块不需要在实训阶段模拟复杂的三维车体振动、轮轨接触、转向架动力学等高级问题。建议范围限定为：

```text
单列车
一维纵向运动
离散牵引/制动级位
基础阻力 + 坡度阻力
可选曲线阻力
制动响应延迟
停车状态判定
运行曲线输出
```

## 4.2 业务目标

车辆动力学模块应回答以下问题：

1. 当前列车在哪里？
2. 当前列车速度和加速度是多少？
3. 给定牵引或制动指令后，下一时刻列车状态如何变化？
4. 列车是否已经停稳？
5. 列车是否越过目标停车点？
6. 车辆运行是否受到供电、坡度、限速、信号约束影响？

## 4.3 来源分析

railML rollingstock 将车辆、车辆族和编组作为 rolling stock 数据交换的核心对象，并明确车辆技术数据可用于制动曲线和运行时间计算。因此，本模块必须维护车辆技术参数和编组参数，不能只维护速度位置。

OpenTrack 类铁路仿真工具通常维护机车车辆技术特性，包括牵引力-速度关系、载重、长度、黏着系数等。这说明车辆动力学模块至少应支持牵引/制动能力随速度和车辆参数变化。

SUMO 铁路仿真文档说明，铁路场景需要网络建模、车辆运行、停车和仿真输出。这支持本模块输出标准运行状态，并与线路、车站、调度、记录模块联动。

EN 50641 虽然是牵引供电仿真工具验证标准，但其中将“列车机械运动计算”和“牵引供电负荷潮流计算”都列为核心仿真函数。这说明车辆运动和供电仿真不能割裂，车辆模块要向供电模块提供功率/能耗相关状态。

## 4.4 需求范围

### 4.4.1 Must 范围

必须实现：

- 列车位置、速度、加速度更新。
- 牵引级位和制动级位输入。
- 运行阻力、坡度阻力的简化计算。
- 速度不允许为负。
- 到达目标停车点附近并速度为 0 时判定停车。
- 输出速度-时间、位置-时间、牵引/制动级位-时间数据。
- 接收供电模块给出的牵引限制比例。

### 4.4.2 Should 范围

建议实现：

- 牵引力-速度曲线。
- 制动力-速度曲线。
- 制动响应延迟。
- 限速约束检查。
- 自动停车所需的制动距离估算。
- 列车运行阶段识别：启动、加速、巡航、惰行、制动、停车。

### 4.4.3 Could 范围

时间允许时实现：

- 黏着限制。
- 曲线阻力。
- 载客质量变化。
- 多编组车辆参数。
- 能耗与再生制动更精细计算。

## 4.6 车辆状态更新模型建议

### 4.6.1 最小模型

仿真步长为 `dt` 秒时：

```text
net_force = traction_force - braking_force - resistance_force - gradient_force
acceleration = net_force / mass
speed_next = max(0, speed + acceleration * dt)
position_next = position + speed * dt + 0.5 * acceleration * dt^2
```

### 4.6.2 阻力模型

最小版本可使用常数阻力：

```text
resistance_force = constant_resistance
```

推荐版本可使用 Davis 形式的简化阻力：

```text
resistance_force = A + B * speed + C * speed^2
```

### 4.6.3 坡度阻力

```text
gradient_force = mass * g * gradientPermille / 1000
```

上坡为正阻力，下坡为负阻力。

### 4.6.4 牵引功率

```text
traction_power_kw = traction_force * speed / 1000
```

低速时可按牵引力限制，高速时可按功率限制。

### 4.6.5 制动和停车判定

停车判定建议：

```text
speedMps <= 0.05
且 brakeLevel > 0 或目标控制要求停车
```

停车合格判定由车站模块完成，但车辆模块需要提供实际停车位置。

## 5. 第3模块：牵引供电仿真模块

## 5.1 模块定位

牵引供电仿真模块用于模拟电气化轨道交通系统中供电网络对列车运行的支持和约束。对于本实训项目，它的重点不是完整电力系统潮流求解，而是：

```text
供电区段状态
电压/电流/功率估算
牵引功率限制
能耗统计
再生制动能量估算
供电故障注入
与车辆模块联动
```

如果后期有足够时间，可以扩展为更接近 OpenPowerNet 的供电网络仿真，但第一版应优先保证能为车辆控制和自动停车提供清晰约束。

## 5.2 业务目标

牵引供电模块应回答以下问题：

1. 当前列车所在供电区段是否可用？
2. 当前可用电压、电流和功率上限是多少？
3. 供电条件是否限制列车牵引能力？
4. 本次运行消耗了多少电能？
5. 制动时是否产生再生制动能量？
6. 供电故障或欠压会如何影响列车运行？

## 5.3 来源分析

EN 50641:2020 明确适用于 AC 和 DC 牵引供电系统仿真工具验证，核心函数包括列车机械运动计算和电气牵引供电负荷潮流计算。这说明牵引供电仿真不应只是一个静态开关，而应至少能根据列车运行状态计算负载和供电输出。

OpenPowerNet 定位为牵引供电和列车性能仿真软件，其技术资料强调 AC/DC 供电网络、电气网络结构和与列车运行仿真的同步协同。这支持本项目设计“供电模块与车辆模块同步交换功率需求和供电限制”。

开放论文关于 DC 第三轨再生制动能量回收的研究说明，再生制动能量可被邻近牵引列车消耗、储能设备吸收或反馈至上级电网。这说明本项目可在增强版本中统计再生能量，不必一开始实现完整回馈网络。

## 5.4 需求范围

### 5.4.1 Must 范围

必须实现：

- 供电区段配置。
- 供电状态：正常、欠压、断电、故障。
- 电压、电流、功率上限字段。
- 根据车辆功率需求估算能耗。
- 根据供电状态输出牵引限制比例。
- 支持供电故障注入。
- 向可视化和数据记录模块输出供电状态。

### 5.4.2 Should 范围

建议实现：

- 再生制动能量估算。
- 区段切换。
- 多种供电故障：欠压、过载、断电、限流。
- 供电恢复。
- 供电事件日志。

### 5.4.3 Could 范围

时间允许时实现：

- 多列车负载叠加。
- 简化直流牵引网络潮流。
- 再生能量被邻近列车吸收。
- 储能设备模型。
- 变电所负载曲线。

## 5.6 供电计算模型建议

### 5.6.1 最小功率估算

车辆模块输出牵引力和速度：

```text
mechanical_power_kw = traction_force_n * speed_mps / 1000
electrical_power_kw = mechanical_power_kw / traction_efficiency
```

如果没有牵引力数据，可由牵引级位近似：

```text
electrical_power_kw = max_power_kw * tractionLevel / maxTractionLevel
```

### 5.6.2 电流估算

```text
current_a = electrical_power_kw * 1000 / voltage_v
```

### 5.6.3 能耗积分

```text
energy_kwh += power_kw * dt_seconds / 3600
```

### 5.6.4 牵引限制比例

```text
if status == OUTAGE:
    tractionLimitRatio = 0
elif status == UNDERVOLTAGE:
    tractionLimitRatio = voltageV / nominalVoltageV
elif requestedPowerKw > maxPowerKw:
    tractionLimitRatio = maxPowerKw / requestedPowerKw
else:
    tractionLimitRatio = 1
```

### 5.6.5 再生制动能量估算

简化估算：

```text
if brakeLevel > 0 and speedMps > 0:
    regen_power_kw = braking_force_n * speed_mps * regen_efficiency / 1000
    regenerative_energy_kwh += regen_power_kw * dt_seconds / 3600
```

第一版可以只统计“理论可回收能量”，不模拟电网是否实际吸收。

## 9. 对本项目的落地建议

## 9.1 第一版实现建议

第一版目标是“能跑、能停、能算、能展示”。

车辆模块：

```text
单列车一维运动
牵引/制动离散级位
基础阻力 + 坡度阻力
目标停车点距离
停车状态判定
运行曲线输出
```

供电模块：

```text
供电区段
标称电压
最大功率
当前功率估算
牵引能耗积分
欠压/断电故障
牵引限制比例
```

第一版验收标准：

- 列车能从起点运行到站点附近。
- 能通过制动停下。
- 能计算停车误差。
- 能显示速度、位置、牵引/制动、功率和能耗。
- 断电故障能限制牵引。

## 9.2 第二版实现建议

第二版目标是“更像真实系统”。

车辆模块：

```text
牵引力-速度曲线
制动力-速度曲线
制动延迟
运行阶段识别
舒适性指标
```

供电模块：

```text
再生制动能量估算
供电故障恢复
电压/电流曲线
供电限制持续时间
```

第二版验收标准：

- 同一控制策略下，不同供电状态影响运行时间和停车效果。
- 能展示能耗曲线。
- 能展示正常运行与故障运行对比。

## 9.3 不建议第一阶段做的内容

以下内容技术价值高，但不适合第一阶段投入太多时间：

- 复杂三维车辆动力学。
- 完整轮轨接触模型。
- 完整牵引供电潮流求解。
- 多列车再生能量实时分配。
- 牵引变电所保护逻辑。
- 谐波、电磁兼容、短路计算。
- 轨电位和杂散电流分析。

这些内容可作为答辩中的“后续扩展方向”。
