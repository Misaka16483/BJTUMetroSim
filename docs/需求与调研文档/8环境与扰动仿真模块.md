# 城市轨道交通环境与扰动仿真模块需求调研文档

## 1. 调研范围说明

本文档聚焦城市轨道交通模拟仿真系统中的**环境与扰动仿真模块**。该模块对应前八个业务模块中的第 8 个模块，主要研究异常、故障、外部条件和客流波动如何影响列车运行、车站服务、调度恢复和系统安全。

根据调研指导要求，本模块最终输出包括：

* 资料来源登记表；
* 需求候选表；
* 数据对象清单；
* 典型仿真场景清单；
* 接口候选表；
* 评估指标清单；
* 需求追踪矩阵。

---

## 2. 模块边界说明

### 2.1 模块名称

环境与扰动仿真模块。

### 2.2 现实对象

该模块模拟城市轨道交通运行中出现的外部环境变化、设备故障、客流冲击、运行延误和突发事件。

### 2.3 输入

* 扰动类型；
* 扰动开始时间；
* 扰动持续时间；
* 扰动影响范围；
* 扰动影响强度；
* 列车状态；
* 车站客流状态；
* 线路与信号状态；
* 调度命令。

### 2.4 输出

* 扰动事件状态；
* 故障触发结果；
* 对列车速度、停站时间、运行间隔、延误的影响；
* 对车站客流密度、站台滞留人数、疏散时间的影响；
* 扰动恢复过程记录；
* 正常场景与扰动场景对比结果。

### 2.5 影响模块

环境与扰动模块会影响：

* 车辆动力学仿真模块；
* 牵引供电仿真模块；
* 信号与列控仿真模块；
* 车站与站台仿真模块；
* 调度与运行图仿真模块。

### 2.6 最小实现范围

本项目最小实现范围为：支持按时间、位置或人工方式注入扰动事件，至少包括**临时限速、车门/屏蔽门故障、供电故障、信号故障、大客流扰动、停站时间延长、火灾/烟雾疏散扰动**，并记录扰动前后的运行时间、停站时间、列车延误、站台客流和恢复过程。

## 5. 现有研究结论提炼

### 5.1 大客流扰动会显著影响停站时间

Jiang 等的研究指出，城市轨道交通停站时间受列车间隔、上下车人数、上下车过程以及列车过度拥挤导致车门首次关闭失败等因素影响。该研究以上海轨道交通 8 号线为案例，说明高峰期客流过大时，原有固定停站时间难以满足实际运行需求。

因此，在仿真系统中，不能只把停站时间设置为固定常数，而应支持“基础停站时间 + 客流附加时间 + 故障附加时间”的组合模型。

### 5.2 停站时间扰动容易引起延误传播

Adachi 等针对东京地铁东西线的研究表明，高密度运营线路中，突发事件、乘客集中上车、车门夹物等会导致停站时间延长，而停站时间一旦超过计划值，延误会快速传播到后续列车。

因此，停站时间扰动应被视为高影响扰动，尤其适用于早晚高峰、小间隔、高满载率线路。

### 5.3 客流扰动和列车运行扰动存在耦合关系

Farhi 等提出的地铁交通控制模型考虑了运行时间、停站时间、安全间隔和乘客需求之间的关系，并指出如果乘客需求对停站时间的影响没有被有效控制，系统运行可能出现不稳定。

Seo 等进一步指出，城市轨道交通中存在列车拥堵和乘客拥堵两类拥堵，这两类拥堵会相互作用并放大延误。

因此，环境与扰动模块不应只记录故障本身，还应把扰动影响传递给车站、列车、运行图和调度模块。

### 5.4 大客流管控措施会改变扰动影响

宋晓敏的研究基于动态路径规划模型，分析了关闭半数进站闸机、关闭朝向站台扶梯、加开空车及组合措施对地铁进站大客流的影响。结果表明，三类措施及其组合均可降低站台密度，其中“闸机 + 扶梯 + 列车”的组合效果最好；单一措施中，加开空车使上车人数较无措施增加 19.07%，乘客平均出行时间降低 15.46%。

因此，系统应支持将“限流、关闭扶梯、加开空车”等作为扰动后的恢复动作或调度响应措施。

### 5.5 调度与客流控制需要联合仿真

Yuan 等针对大规模城市轨道交通网络研究列车调整和客流控制策略，指出突发扰动和高峰需求会导致列车延误、站台拥挤和运行不稳定。该研究表明，列车运行调整和乘客流控制应联合考虑。

因此，环境与扰动模块应向调度与运行图模块输出扰动后的延误状态、站台拥挤状态和恢复需求。

### 5.6 火灾和烟雾属于低频高后果扰动

Zhang 等基于 BIM、PyroSim 和 Pathfinder 研究地铁站火灾烟气扩散和人员疏散，指出地铁站作为封闭空间，火灾发生时需要分析烟气扩散和疏散能力。

因此，火灾、烟雾和疏散类扰动虽然不是常规运行扰动，但应作为应急仿真和安全评估的重点场景。

## 6. 扰动类型分类

| 扰动大类   | 具体扰动                   | 主要影响对象   | 典型影响             |
| ------ | ---------------------- | -------- | ---------------- |
| 客流扰动   | 进站客流突增、换乘客流突增、站台滞留     | 车站、站台、列车 | 停站时间增加、站台拥挤、列车满载 |
| 停站扰动   | 上下车时间延长、车门反复关闭失败、屏蔽门故障 | 车站、列车、调度 | 到发时刻偏差、后续列车延误    |
| 列车运行扰动 | 初始晚点、区间停车、运行时间波动       | 列车、运行图   | 运行间隔不均、延误传播      |
| 信号扰动   | 信号故障、移动授权异常、限速命令       | 信号、车辆、调度 | 限速、停车、发车间隔扩大     |
| 供电扰动   | 供电中断、欠压、牵引功率受限         | 供电、车辆    | 牵引能力下降、运行时间增加    |
| 设备扰动   | 车门故障、屏蔽门故障、扶梯停运、闸机故障   | 车站、站台、乘客 | 服务能力下降、拥堵加剧      |
| 环境扰动   | 雨雪、低粘着、站内积水、能见度下降      | 车辆、车站、乘客 | 制动距离变化、疏散效率下降    |
| 突发事件   | 火灾、烟雾、异物侵限、乘客跌落轨道      | 全系统      | 局部停运、疏散、应急调度     |

## 15. 可采用的简化建模方法

### 15.1 停站时间修正模型

```text
实际停站时间 = 基础停站时间 + 客流附加时间 + 车门故障附加时间 + 屏蔽门故障附加时间
```

其中：

```text
客流附加时间 = k1 × 上车人数 + k2 × 下车人数 + k3 × 站台密度修正系数
```

### 15.2 临时限速扰动模型

```text
当列车位置 ∈ [限速起点, 限速终点] 且扰动状态 = 生效：
    当前允许速度 = min(线路限速, 信号限速, 临时限速)
```

### 15.3 供电扰动模型

```text
可用牵引力 = 正常牵引力 × 牵引限制比例
```

例如：

```text
正常状态：牵引限制比例 = 1.0
欠压状态：牵引限制比例 = 0.5
供电中断：牵引限制比例 = 0
```

### 15.4 延误增量模型

```text
单车延误增量 = 扰动场景实际到达时间 - 正常场景实际到达时间
总延误增量 = 所有受影响列车延误增量之和
```

### 15.5 恢复时间模型

```text
恢复时间 = 系统重新满足以下条件的时间 - 扰动开始时间

恢复条件：
1. 扰动状态为已结束；
2. 列车运行间隔恢复到允许范围；
3. 站台密度低于安全阈值；
4. 后续列车到发偏差低于设定阈值。
```

## 16. 结论

现有研究表明，城市轨道交通扰动的影响通常不是孤立发生的，而是通过“客流变化—停站时间变化—列车间隔波动—延误传播—站台积聚”的链式机制不断放大。大客流、停站时间延长、车门/屏蔽门故障、信号故障、供电故障、临时限速和火灾烟雾是环境与扰动仿真模块中最值得重点建模的扰动类型。

本项目在实现时可采用“事件注入 + 影响传播 + 指标对比”的方式：先定义扰动事件，再把扰动影响传递给车辆、车站、信号、供电和调度模块，最后通过运行时间、停站时间、延误增量、站台密度和恢复时间等指标判断仿真是否合理。
