# 客流数据合理性审查与校准 V1

## 1. 结论

原客流配置适合演示泊松到达、上下客和载荷闭环，但不适合作为北京地铁9号线的定量实验基线。主要问题不是泊松生成器本身，而是输入数据缺乏版本、来源、口径和量级约束：

1. 26个方向站台基准率合计为 `1670 pax/min`。按现有六时段系数积分，普通工作日理论到达量约 `1,113,222` 人，明显高于项目现有公开资料给出的9号线量级。
2. 北京西站站区六站只占理论总量约 `51.5%`，而北京市应急管理局公开资料给出的口径是该站区日均客运量约40万人次、占9号线70%以上。
3. 郭公庄下行、国家图书馆上行仍生成候车乘客，但这两个方向在对应终点没有继续前进的正线列车，候车量会永久累积。
4. 终点下车比例不是100%，会使到达终点的列车保留不应继续存在的合成乘客。
5. 所有站台都使用 `120 m²`，使北京西站等枢纽和普通站的密度不可比较。
6. 场景列车定员为600人，与项目API和总体设计中使用的1460人不一致，导致满载率、滞留、车辆质量和供电负荷的实验口径冲突。
7. 主引擎、独立客流仿真和演示脚本分别维护不同的站级需求，形成多个数据源。

本次将配置升级为 `LINE9-CALIBRATED-SYNTHETIC-V1`。它仍是合成数据，不是AFC或OD实测，但具备明确的聚合量级锚点、方向潮汐、终点约束、版本和质量标签。

## 2. 校准依据与口径限制

北京市应急管理局2022年公开材料指出，北京西站站区所辖国家图书馆、白石桥南、白堆子、军事博物馆、北京西站和六里桥东6站，日均客运量约40万人次，占9号线总客运量70%以上；北京西站最高单日进出站量达31.8万人次。[北京市应急管理局资料](https://yjglj.beijing.gov.cn/art/2022/12/20/art_9408_638032.html)

该资料的“客运量”和本模型的“进入方向站台的合成到达人数”不是完全相同的统计口径，因此这里只作为量级和空间占比锚点，不能宣称复原了真实进站量、换乘量或OD。配置质量明确标记为 `CALIBRATED_SYNTHETIC`。

北京西站还受国铁到达脉冲影响。北京市交通委员会2024年资料给出暑运期间北京西站日均到客14.5万人次、周日最高17.8万人次，说明仅用平滑六时段曲线无法描述重点日期的短时冲击。[北京市交通委员会资料](https://jtw.beijing.gov.cn/xxgk/xwfbh/202407/t20240726_3760630.html)

## 3. 校准结果

| 指标 | 原配置 | V1配置 |
| --- | ---: | ---: |
| 工作日理论到达量 | 1,113,222 | 549,986 |
| 北京西站站区六站理论到达量 | 573,276 | 400,059 |
| 北京西站站区占比 | 51.5% | 72.7% |
| 无效终点方向到达率 | 非零 | 0 |
| 终点清客比例 | 5%～25% | 100% |
| 早晚方向潮汐 | 无 | 上行早峰增强、下行晚峰增强 |
| 站台有效面积 | 全站120 m² | 100～240 m²分站工程假设 |
| 默认列车定员 | 600 | 1460 |
| 数据质量标签 | 无 | `CALIBRATED_SYNTHETIC` |

计算仍使用既有六时段系数，以保持运行图时段边界兼容。每个方向增加时段因子：早高峰上行 `1.15`、下行 `0.85`，晚高峰反向；晚间下行略高。站级基准率总和从1670降到825，并把需求权重集中到公开资料覆盖的六个北部枢纽站。

## 4. 实现变化

- 新增版本化配置：`data/passenger_profiles/line9_calibrated_synthetic_v1.json`。
- 新增加载与质量校验：`app/domain/station/passenger_profiles.py`。
- 主引擎与独立客流仿真读取同一配置，消除重复数据源。
- `StationFlowConfig`支持分方向时段因子和分站有效候车面积。
- 主引擎快照KPI增加 `passengerProfileId`、`passengerDataQuality` 和 `estimatedWeekdayPassengerArrivals`，使实验结果可追溯。
- 单车和五车场景的定员统一为1460；五车初始载客数按原负载率等比例换算。
- 新增 `tools/audit_passenger_profile.py`，可重复核对理论日量、站区占比和终点约束。

运行审计：

```powershell
python tools/audit_passenger_profile.py
```

## 5. 仍不能据此得出的结论

V1配置不能用于声称真实站间OD、真实断面满载率或真实换乘比例。固定下车比例仍然只是过渡模型；虽然终点强制清客，但中间站并不按乘客目的地下车。下一阶段应采用已有《9号线OD矩阵与换乘客流调研报告》的方案，引入5分钟OD、乘客批次和换乘边界节点。

此外，`doorCapacityPaxPerSec`目前在一次性停站服务中生效，但主引擎连续上下客路径没有以同一方式限制总车门吞吐，也没有根据未完成上下客自动延长停站。这是运行逻辑缺口，不应通过继续放大或缩小客流率掩盖。

站台有效面积、1460人定员、65 kg平均乘客质量和停站参数仍属于工程假设。获得车辆技术资料、站台有效候车面积、AFC和列车运行日志后，应分别替换并保留新版本，不能覆盖V1以维持实验可复现性。
