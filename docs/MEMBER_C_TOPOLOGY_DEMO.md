# 成员 C：联锁拓扑检验演示

## 页面定位

主前端启动后，在顶部导航的 `TEST` 页面可以进入本演示；也可以直接访问后端的
`/api/phase2/member-c/demo`。该页面是一个独立的成员 C 联锁检验会话，**不会与主
仿真引擎中的多列车状态同步**。

本页呈现的是基于线路数据、进路表和道岔关系生成的**逻辑拓扑图**，不是原有线路
图的物理复刻：横向和纵向位置只服务于阅读连接关系、减少交叉和标签重叠，不能表示
实际里程、实际站场尺寸或轨道几何形状。所有 Seg、信号机、道岔和进路关系均应以
线路数据与进路表为准。

## 图形含义

- `S编号`：一个轨道 Seg（逻辑轨道区段）。点击后，右侧“轨道与进路”会列出经过该
  Seg 的进路；选择进路后，该进路包含的 Seg 会高亮。
- 信号机：按所属 Seg、偏移位置和方向画在轨道一侧；灯色来自当前演示会话的联锁状态。
  红灯表示未获准通过；黄灯表示已办理本进路、但终端信号仍为红，列车可以进入但必须
  准备在终端前停车；绿灯表示前方连续进路已办理且畅通。
- 道岔：岔心 Seg 旁标有道岔编号及当前位置。灰色连接表示普通连接或道岔定位方向，
  琥珀色连接表示道岔反位的分歧腿。连接颜色描述的是拓扑/道岔关系，不是信号灯色。
- 半透明绿色框：该 Seg 关联站台。
- 红色轨道：检验小车的车体覆盖范围，即当前被占压的 Seg。车辆位置在演示中按沿已选
  路径的连续距离计算，再映射到 Seg 与 Seg 内偏移位置。
- 青绿色虚线覆盖层：当前已锁闭的进路。它与红色占压、黄色道岔分歧相互独立。

## 自动演示

页面初始会启动一列检验车，并沿预先从进路表关联出的连续进路链运行。上方按钮提供：

- `Play`：恢复自动运行。
- `Pause`：停止自动推进 tick，保留当前联锁、占压和灯色状态。
- `Step`：只执行一个 0.5 秒 tick，便于逐步检查列车位置、占压、进路锁闭与释放。
- `Reset`：重新初始化本演示会话。

自动模式会在前方保持有限的连续进路预告量，以便展示绿、黄、红三种灯色的差异；它
不是完整的行车调度或 ATO 路径选择器，遇到分支时不会根据运营计划自动决定下一条支路。

## 手动办理与冲突验证

1. 在右侧切换到“轨道与进路”，点击图上的一个 Seg，查看关联进路。
2. 对某条进路使用“在始端放置并办理”，会新建一轮手动检验会话：在该进路始端信号前
   放置小车，并请求锁闭该进路。
3. 要检验第二条进路是否与当前锁闭进路冲突，使用目标进路的“仅办理（不换车）”。它会
   保留现有小车和锁闭状态，因而可正确触发敌对进路或区段占用检查。
4. 若只想检查某一 Seg 的占压影响，可先点击它并使用“在 S编号 起点放置小车”，再办理
   相关进路。

办理成功时，进路会显示青绿色虚线并锁闭所需区段和道岔。冲突或不具备条件时不会覆盖
原有锁闭，常见失败原因包括“存在敌对进路已锁闭”“进路区段当前被占用”和“所需道岔
不可用”。

## 日志查看

右侧“事件日志”按 tick 显示中文事件，包括小车放置、进路锁闭、区段占压、列车通过、
进路释放和办理失败。可借此核对一次操作是否真正进入联锁流程：先确认“办理成功，已
锁闭道岔和区段”，再观察锁闭/释放和占压事件；冲突验证应出现明确的失败原因，且原有
锁闭进路仍保留在图上。
