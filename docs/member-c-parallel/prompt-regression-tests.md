# 回归测试线程提示词

你负责为本轮修复建立独立、可重复的验收测试。先阅读 `docs/member-c-parallel/README.md`。

严格限制：不得修改任何生产代码；尽量只新增测试文件，不编辑既有测试文件。

建立以下测试：

1. 七里庄上行到六里桥东，规划为平台 12 到平台 14，实际到达 `S103`。
2. 到站 tick 不得从 `S103`跳到 `S88`。
3. 进路 50 必须观察到区段 `88/S103`被所属列车占压；车体尚未出清时保持锁闭，车尾出清后释放。
4. 六里桥东下一程必须从实际到达平台 14、`S103`开始，而不是重新选择平台 13、`S88`。
5. 普通中间站没有调度任务时不得隐式换向或渡线。
6. 连续车体路径跨越前后 PathPlan 时，旧路径上的车尾占压不能提前消失。
7. 道岔分支只占压实际经过的分支。
8. 郭公庄和国家图书馆折返不得瞬移；方向只能在配置的折返点、速度为零时改变。
9. 折返各阶段必须经过进路锁闭、MA、实际 Seg 运动和车尾释放。

测试可以依据 `README.md`约定的目标接口编写，并允许在生产实现合入前失败；但失败原因必须是待实现行为，而不是测试自身语法、导入或不稳定计时。优先使用确定性逐 tick 仿真，不依赖真实时间等待。
