# 折返规划线程提示词

你负责任务感知的进路选择策略和终点折返方案生成。先阅读 `docs/member-c-parallel/README.md`。

严格限制：不要修改 `app/core/engine.py`，不要修改 `SectionOccupationService`。

任务：

1. 记录现有 `MainRouteFirstPolicy`的行为：主进路优先、进路数量更少优先、稳定 ID 优先。
2. 引入明确的运行任务语义，至少支持 `NORMAL`、`TURNBACK`、`TRANSFER`、`DEPOT`。
3. 普通客运候选必须以实际平台为起点；无调度任务时，隐式换向应禁止，渡线应排除或处于最低优先级。
4. 终点折返应优先选择配置允许的折返方案，但不能把折返简化为一条单方向站台到站台链。
5. 使用真实进路、信号、Seg 和道岔数据分析郭公庄、国家图书馆，识别“驶向折返点 -> 停车换端 -> 反向驶入发车轨”的多阶段计划。
6. 若原始数据没有权威的折返语义标记，建立小型、可验证的终点折返配置。配置可以引用进路 ID、折返点和目标平台，但算法不得把路线 ID 散落硬编码在逻辑中。
7. `TurnbackPlan`应包含有序阶段、每阶段方向、真实进路链、路径、折返点和最终平台，并能由现有 `RouteCatalog`验证。
8. 增加独立测试：普通中间站不隐式换线；六里桥东从平台 14 继续规划；两个终点产生合法多阶段折返计划；非法或缺失配置明确失败。

注意：终点站台对目前无法由现有规划器直接得到候选，必须支持中途方向反转或显式阶段配置。不要通过修改平台 ID 或 Seg 实现换线。

完成后运行聚焦测试并提交，不推送。报告提交哈希、发现的终点进路组合、测试结果，以及主引擎需要实现的状态转换接口。
