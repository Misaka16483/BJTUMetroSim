# 主集成线程提示词

你负责成员 C 修复的主引擎集成。先阅读 `docs/member-c-parallel/README.md`，严格遵守其中的职责边界。

你的首要任务不是终点折返，而是先根治普通中间站到站不连续：

1. 为 `SimTrainState`增加正式的 `current_platform_id`，并在快照中公开。
2. 加车、规划、到站时维护真实平台身份。
3. 到站时保存本次 `PathPlan.destination_platform_id`，不得调用按方向猜测平台的逻辑覆盖它。
4. 保留完成路径和终点位置，至少让移动后的联锁扫描观察到真实目的 Seg。
5. 下一程规划必须以 `current_platform_id`作为唯一候选起点；找不到合法进路时停车重试，不得回退到整站任意平台。
6. 普通中间站禁止隐式换线和位置瞬移。
7. 为七里庄到六里桥东补充集成回归，确认进路 50 先观察到区段 88，随后在车尾出清后正常释放。

开始前复现并记录当前行为。修改应保持 `RouteService`为进路生命周期唯一权威，不要在主引擎里直接释放进路。

占压路径和折返规划由其他线程实现。在它们交付前，只定义最小集成接口，不重复实现其内部逻辑。后续按顺序 cherry-pick，并由本线程独占修改 `app/core/engine.py`。

每阶段完成后运行聚焦测试并提交；最后运行完整后端测试和前端构建。不要推送远端，不要拉取或合并新的 `main`。
