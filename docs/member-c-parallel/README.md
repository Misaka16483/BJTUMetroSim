# 成员 C 并行修复协作说明

## 目标

修复列车到站后平台和轨道位置不连续、进路 50 无法完成生命周期、道岔区域车尾占压路径不准确，以及终点折返依赖瞬移的问题。

## 共同原则

- 主引擎是列车运动状态的唯一权威。
- `RouteService` 是进路进入、逐段解锁和最终释放的唯一权威。
- ATO 只提供牵引/制动控制，不申请或释放进路。
- 普通客运运行必须从列车实际所在平台规划，不得退回整站任意平台。
- 所有跨轨移动必须经过真实 Seg、进路、道岔、MA 和车辆运动，禁止直接修改位置实现换线。
- `PathPlan` 与实际申请的进路链必须来自同一个 `RouteChainPlan`。
- 并行开发期间不合并新的远端 `main`。

## 已确认的复现事实

- 七里庄到六里桥东上行使用平台 12 到平台 14、进路 `49 -> 50`、Seg `S98 -> ... -> S103`。
- 到站后主引擎把列车重新锚定到平台 13、`S88`。
- 进路 50 因未观察到本车占压最后区段 `88/S103`，停留在 `APPROACH_LOCKED`。
- 从平台 14、`S103`到下一站存在合法进路链；问题是现有规划重新枚举整站平台后选了从 `S88`开始的候选。
- 郭公庄和国家图书馆的站台之间没有可由现有单方向站台到站台规划器直接得到的折返链。终点折返需要多阶段进路和中途换向。

## 目标接口

最终集成至少应具备以下显式概念，具体模块可以在保持职责边界的前提下调整：

- `current_platform_id`：列车当前实际所在平台，随真实运行更新。
- `OperationIntent`：至少包含 `NORMAL`、`TURNBACK`、`TRANSFER`、`DEPOT`。
- `TurnbackPlan`：由一段或多段真实进路、折返点、每段方向和目标平台组成。
- `TrainTrackTrace`：保存足以覆盖整列车长度的连续实际行驶 Seg 走廊，跨越前后两个 PathPlan。

## 集成顺序

1. 主线程先修复普通到站平台与位置连续性。
2. 接入连续车体路径和路径感知占压。
3. 接入任务感知的选择策略和终点折返规划。
4. 实现主引擎折返状态机。
5. 合入完整回归测试并执行全套验证。
