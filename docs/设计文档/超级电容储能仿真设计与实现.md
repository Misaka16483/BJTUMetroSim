# 超级电容储能仿真设计与实现

## 1. 目标与边界

本设计在既有 DC750V 牵引供电潮流中增加地面超级电容储能系统（SCESS），形成以下能量闭环：

1. 列车再生能量优先供本车辅助负载；
2. 通过可达供电路径供其他牵引列车；
3. 剩余再生能量给超级电容充电；
4. 超级电容无法吸收的部分送能量反馈装置；
5. 最终无法利用部分记为再生未利用功率；
6. 后续列车牵引时，超级电容优先放电，再由整流机组补足。

超级电容是独立设备，不复用牵引所的 `efsCapacityKw`。后者只表示向交流侧反馈的可逆变流容量。

本阶段采用准静态等效模型，不模拟单体电芯、电化学温升和开关器件 PWM。设备参数均为工程估算，不代表北京地铁9号线竣工配置。

## 2. 工程依据

- ABB Enviline ESS 750 产品公开量级为 1.35/2.7MW、最高25.3kWh，证明“兆瓦级功率、数十kWh能量”适用于750V轨道供电。
- 北京某750V地铁研究采用每隔一座牵引所配置14kWh超级电容，SOC工作区间0.25至1，并联合优化位置、容量和控制参数。
- 香港南港岛线在六处供电设施中选择两处配置CAPAPOST，说明储能通常按关键位置配置，而非每座牵引所等量安装。

## 3. 设备模型

超级电容储能量满足：

```text
E = 0.5 * C * Usc^2
SOC = E / Erated
```

离散状态更新为：

```text
E(t+dt) = E(t)
          + etaCharge * Pcharge * dt
          - Pdischarge / etaDischarge * dt
          - Pstandby * dt
```

充放电功率同时受变流器额定功率和SOC剩余空间限制：

```text
Pcharge <= min(PchargeRated, (Emax-E)/(etaCharge*dt))
Pdischarge <= min(PdischargeRated, (E-Emin)*etaDischarge/dt)
```

SOC达到上下限后不再继续充放电。设备所在牵引所停运时，SCESS同步退出运行。

## 4. V1场景参数

| 参数 | SCESS-0905 |
|---|---:|
| 接入位置 | TS-0905 |
| 额定储能量 | 20kWh |
| 最大充电功率 | 2000kW |
| 最大放电功率 | 2000kW |
| 放电削峰触发阈值 | 1000kW |
| 初始SOC | 50% |
| 最低/最高SOC | 20%/90% |
| 充电/放电效率 | 95%/95% |
| 待机损耗 | 3kW |
| 数据质量 | ENGINEERING_ESTIMATE |

选择TS-0905只是用于全线中部演示，不是选址结论。后续实验必须比较无储能、单点储能和多点分布式储能。

放电控制采用基础削峰策略：只有设备电气可达范围内的牵引需求超过 `dischargeTriggerPowerKw`
时才放电，放电目标为超出阈值的部分。这避免辅助负荷或低负荷持续消耗有限的储能，
并使超级电容服务于削减整流机组峰值功率。

## 5. 潮流与能量守恒

求解器按电气可达性和路径电阻分配功率。超级电容充电路径计入再生传输损耗，放电路径计入储能传输损耗。DC母线侧功率平衡为：

```text
整流输入 + 已接受再生 + 储能放电
= 列车实际用电 + 线路损耗 + 交流反馈 + 储能充电
```

充放电效率损耗发生在储能内部，通过 `conversionLossesKw` 单独统计，不重复计入DC母线潮流损耗。

## 6. 数据契约

静态拓扑新增：

```text
supercapacitorStorageSystems[]
  storageId, substationId
  ratedEnergyKwh
  maxChargePowerKw, maxDischargePowerKw
  dischargeTriggerPowerKw
  initialSoc, minSoc, maxSoc
  chargeEfficiency, dischargeEfficiency
  standbyPowerKw, status, quality, parameterSources
```

动态快照新增：

```text
supercapacitorStorageSystems[]
  soc, storedEnergyKwh
  availableChargeEnergyKwh, availableDischargeEnergyKwh
  chargePowerKw, dischargePowerKw, conversionLossesKw
  cumulativeChargedKwh, cumulativeDischargedKwh
  state, status
```

再生汇总新增 `storageChargedKw` 和 `storageDischargedKw`。再生路径中的 `sinkType=SUPERCAPACITOR`表示列车到储能的充电路径。

运行记录新增 `supercapacitor_storage_systems` 静态设备表和
`supercapacitor_power_records` 动态功率表，用于回放 SOC、充放电功率、变换损耗和累计能量。

## 7. 可视化

供电专用页面增加：

- 拓扑图SC标记和实时SOC；
- 储能充电、放电瞬时指标；
- 充放电功率历史曲线；
- 储能设备表，包括SOC、能量、功率、损耗和状态；
- 再生能量路径中的超级电容去向。

颜色约定：绿色充电、橙色放电、蓝色待机。

## 8. 验收与后续深化

当前自动化验收覆盖：

- 制动时能够充电且再生未利用功率低于无储能基线；
- 后续牵引时能够放电；
- SOC严格保持20%至90%；
- 超级电容与能量反馈装置字段彼此独立；
- 潮流平衡误差率小于1%；
- 既有多车、故障、联络开关和再生守恒测试不回归。
- 多车同时制动时，单台储能的聚合充电功率不超过设备额定值；
- 低于削峰阈值的基础负荷不触发放电，高峰负荷才使用储能；
- 无效求解结果回滚 SOC，供电网络复位原子清除储能瞬态与累计量；
- 运行记录和导出包含超级电容的逐步状态。

后续优化变量包括安装位置、设备数量、额定能量、额定功率、SOC范围和充放电阈值；目标包括电网取电量、再生未利用量、最大需量、最低网压、线路损耗、设备投资和等效循环次数。建议先采用NSGA-II进行离线选址定容，再研究带硬约束的在线控制算法。
