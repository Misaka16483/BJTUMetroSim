# 9号线全线联锁图信号机文档

> 生成日期：2026-07-11 | 数据来源：`data/cache/line_map.json`（信号机表 157 条）| 对应代码：`bj-metro-sim/src/components/FullLineInterlockingView.tsx`

## 1. 概述

全线联锁图覆盖 9 号线郭公庄（GGZ）→ 国家图书馆（GTG）共 13 座车站。信号机分为两类展示区域：

- **车站信号机**：由各站 `XxxInterlockingData()` 定义，直接在站内轨道上绘制
- **区间信号机**：从 `trackMap.signals` 中按区间 segment ID 过滤，在 `drawConnectionProfile()` 中绘制在连接轨道（`conn-up-{i}` / `conn-dn-{i}`）上

信号机按类型使用不同符号：

| 类型值 | 名称 | 符号 | 颜色 |
|--------|------|------|------|
| 1 | 主信号 | ◆ | `#58a6ff` |
| 2 | 调车信号 | ◇ | `#d29922` |
| 3 | 预告信号 | ● | `#8b949e` |

方向定义：**0xaa = UP（GGZ → GTG）**，**0x55 = DN（GTG → GGZ）**。

## 5. 渲染实现概要

### 5.1 车站信号

由各站 `stationInterlockingData.ts` 中的 `signals` 数组定义，`getCombinedInterlockingData()` 合并时加上站偏移量，在 `drawSignal()` 中绘制。

### 5.2 区间信号

在 `drawConnectionProfile()`（`FullLineInterlockingView.tsx` 第 474 行起）中处理：

1. 根据 `connTrack.segmentIds` 遍历区间所有 segment
2. 从 `trackMap.signals` 中过滤 `signal.segmentId` 在区间 seg 列表中的信号机
3. 计算信号在区间内的绝对里程：`absPos = cumSegStart + signal.offsetM`
4. 通过线性映射 `mileToX(m) = gapX1 + (m / totalLen) * gapW` 转换为画布坐标
5. 绘制竖线 + 符号 + 名称（间距 > 40px 时显示名称）

### 5.3 关键映射

```
画布像素 X = connTrack.x + (信号绝对里程 / 区间总里程) × connTrack.width
```

信号绝对里程 = 该信号所在 segment 之前的所有 segment 长度之和 + `signal.offsetM`
