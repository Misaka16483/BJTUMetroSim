# 全线连续联锁图设计方案

## 1. 设计目标

将当前独立的四个站台联锁图合并为一张连续的全线联锁图，实现：

1. **13站连续展示** — 郭公庄到国家图书馆在一张图上无缝连接
2. **自动定位列车** — 根据 `segmentId + offsetM` 直接定位到火车图标位置
3. **段落等比例缩放** — 站内段宽（展示细节）↔ 区间段窄（保留信息），全程无信息丢失
4. **全线缩放/拖拽** — 放大看站内细节，缩小看全局态势

## 2. 数据基础

后端 `line_map.json` 提供了全线拓扑数据：

| 数据 | 数量 | 用途 |
|------|------|------|
| 主线 segments（沿 `endForwardSegId` 串联） | 76 段 | 铺排轨道的基本单元 |
| 站台 platforms | 56 条 | 标注车站位置、站台边界 |
| 信号机 signals | 157 | 按 segment + offset 精确定位 |
| 静态限速 speedRestrictions | 238 | 在轨道上叠加限速色条 |
| 坡度 gradients | 182 | 在轨道上叠加坡度箭头 |
| 道岔 switches | 60 | 标记道岔分叉位置 |
| 进路 routes | 249 | 进路高亮显示 |

## 3. 布局策略

全线采用分段不等宽布局，按区域类型分配宽度：

### 3.1 区域划分

```
区域类型    宽度(px)    包含内容
─────────────────────────────────────────────
站内段      60~100     站台、两端道岔、信号机、进路
区间段      20~40      限速色条、坡度箭头、段ID
过渡段      40~60      道岔、信号机
```

### 3.2 判断规则

```
┌─ 站区（~600px） ───────┐─ 区间 ─┐─ 下一个站区 ──────┐
│══════╤══╤══╤══╤══╤═══  │ ═════  │  ════╤══╤══╤══╤══│
│  ██站台██ △ ◆ ◆       │ ██▒▒   │     ◆ ◆ △ ████  │
│  seg 11-14            │ seg 19 │    seg 20-23     │
└────────────────────────┘        └──────────────────┘
```

- **站内段**：站台 segment 及其前后各 2-3 段（道岔区域），宽度放大
- **区间段**：远离车站的纯连接段，宽度压缩
- **限速/坡度**：无论段宽如何，色条和箭头始终正确显示在对应比例位置

### 3.3 定位算法

```
给定: segmentId, offsetM
定位: 所在段的 x 坐标 + (offsetM / lengthM) * 段宽
        = 火车在联锁图上的 px 位置
```

### 3.4 参考依据

- 中国传统铁路信号计算机联锁系统的站场图采用站内 1:1、区间压缩的显示策略
- CTC（调度集中）系统的全线图采用不等比例铺排
- 地铁综合监控系统（ISCS）的轨道图也采用站内/区间分比例显示

## 4. 行车定位功能

### 4.1 数据来源

轮询 `GET /api/sim/state` 或订阅 `WS /api/sim/stream`：

```json
{
  "trains": [
    {
      "trainId": "T0901",
      "segmentId": 129,
      "offsetM": 30.0,
      "speedMps": 8.2,
      "direction": "UP"
    }
  ]
}
```

### 4.2 定位流程

```
1. segmentId → 查所属区段
   ├─ 属于某站区 → 在站区内按 offsetM/lengthM 计算 x
   └─ 属于区间 → 直接按区间内比例计算 x
2. 自动滚动视口使火车居中
3. 火车图标（位于公共/metro_train.png）跟随缩放平移
```

## 6. 与现有系统的关系

| 模块 | 关系 |
|------|------|
| `app/api_server.py` | 保持现有 `/api/lines/9/track-map` 端点不变，新增或扩展 `/api/sim/state` 返回完整列车位置 |
| `app/domain/` | 无需新增后端逻辑 |
| `StationInterlockingView.tsx` | 保留现有实现，新增 `FullLineView.tsx` 组件 |
| `stationInterlockingData.ts` | 新增 `fullLineInterlockingData()` 函数 |
| `types/interlocking.ts` | 接口不变，复用现有 `StationInterlockingData` 类型 |

## 7. 分段实施

| 阶段 | 内容 |
|------|------|
| **1. 主链铺排** | 遍历 76 段主线，按站内/区间分比例布局，segment 按 `lengthM` 定宽 |
| **2. 站台标注** | 根据 `platforms` 数据自动标注 13 个车站位置和站名 |
| **3. 信号机定位** | 全线 157 信号机按 segment + offset 精确定位 |
| **4. 限速/坡度** | 叠加全线限速色条和坡度箭头 |
| **5. 列车定位** | 接入后端 API，segmentId + offsetM → 自动定位 + 平滑跟随 |
| **6. 交互增强** | 点击任意 segment 显示详情、车站跳转缩放 |

## 8. 预期效果

- 13站全线在一张横轴上，水平拖拽/缩放浏览
- 火车图标在轨道上跟随实时数据移动
- 信号机按真实灯色显示，区段占用自动高亮
- 站内有道岔/信号细节，区间有限速/坡度，无信息丢失
