# 车辆动力学-供电功率闭环设计与验收

## 1. 目标与边界

车辆模块根据控制指令、速度和老师提供的车辆曲线，计算每列车、每仿真步的候选牵引力、电制动力和网侧功率需求。供电模块统一求解同一仿真步全部列车的 DC750V 潮流，决定实际牵引交付能力和再生能量去向，并将逐车限制反馈车辆模块。

职责边界固定为：车辆提出需求，供电决定交付；车辆不得提前将全部候选再生功率抵扣为已利用能量，供电不得再次通过 `F*v/eta` 覆盖车辆曲线功率。

## 2. 老师曲线口径

数据来源为 `docs/列车仿真参数.xlsx`：

| 数据 | 尺度 | 运行时处理 |
|---|---|---|
| 牵引/制动转矩曲线 | 单电机，整车16台 | 结合齿轮比、传动效率和轮径换算整车轮周力 |
| 网流曲线 | 整车网侧电流 | 用于校核功率尺度，不乘电机数量 |
| 车辆能耗曲线 | 整车0.1秒采样能量 | 除以0.1秒折算kW，不乘电机数量 |

牵引曲线在峰值附近满足 `271 / 0.1 ~= 3277.5 * 825 / 1000 kW`，作为0.1秒尺度和整车尺度的交叉校核。该解释属于基于原表数值关系的工程判定，后续取得老师的字段说明后应更新数据来源等级。

## 3. 每tick接口

车辆侧输出 `VehiclePowerDemand`：

| 字段 | 单位 | 含义 |
|---|---:|---|
| `tractionPowerRequestKw` | kW | 供电限制前的整车牵引取电请求 |
| `regenPowerAvailableKw` | kW | 供电限制前的候选再生功率，只对应电制动力 |
| `auxiliaryPowerKw` | kW | 本车辅助负荷 |
| `tractionMechanicalPowerKw` | kW | 牵引轮周机械功率，诊断字段 |
| `electricBrakeMechanicalPowerKw` | kW | 电制动机械功率，诊断字段 |

供电侧逐车返回：

| 字段 | 含义 |
|---|---|
| `tractionPowerDeliveredKw` | 本tick实际可交付牵引功率 |
| `regenPowerSelfConsumedKw` | 再生能量被本车辅助负荷就地利用的功率 |
| `regenPowerExportedKw` | 扣除本车自用后注入接触网的功率 |
| `regenPowerAcceptedKw` | 本车自用与接触网实际接受功率之和 |
| `regenPowerWastedKw` | 无法利用、需电阻或空气制动替代的功率 |
| `tractionLimitRatio` | 实际牵引能力/请求牵引能力 |
| `regenLimitRatio` | 实际可维持电制动能力/候选电制动能力 |

## 4. 同步顺序

1. 客流更新载客量和车辆质量。
2. ATO或司机控制器生成逐车牵引/制动指令。
3. 动力学根据老师曲线生成候选力和 `VehiclePowerDemand`，暂不推进位置。
4. 引擎一次性提交全部列车负荷，统一求解供电潮流。
5. 供电返回逐车牵引与再生限制。
6. 车辆按牵引限制修正牵引力；按再生限制计算实际电制动力，不足制动力由空气制动补足。
7. 推进车辆状态并累计逐车能量账本。

禁止逐列车“求解一次供电、推进一次车辆”，否则结果会依赖列车遍历顺序。

## 5. 再生分账与守恒

本车自用和接触网注入为：

```text
selfConsumed = min(regenAvailable, auxiliary)
regenExported = max(regenAvailable - auxiliary, 0)
gridDemand = tractionRequest + max(auxiliary - regenAvailable, 0)
```

系统每tick应满足：

```text
regenGenerated
= regenSelfConsumed
 + regenAbsorbedByTrain
 + regenFeedbackToSubstation
 + regenTransferLosses
 + regenWasted
```

逐车应满足：

```text
regenPowerAvailableKw
= regenPowerAcceptedKw + regenPowerWastedKw
```

其中 `regenPowerAcceptedKw` 按再生源侧计量，包含传输过程中产生的线路损耗；系统展示另行给出到达受端的吸收功率和传输损耗。

## 6. 累计指标

每列车累计：`tractionEnergyKwh`、`auxiliaryEnergyKwh`、`regenGeneratedKwh`、`regenSelfConsumedKwh`、`regenAcceptedKwh`、`regenWastedKwh`。

兼容字段 `energyKwh` 定义为车辆实际净电能：

```text
energyKwh += (tractionPowerDeliveredKw + auxiliaryPowerKw - regenPowerAcceptedKw) * dt / 3600
```

候选再生但未被接受的部分不得抵扣净能耗，空气制动不得计入再生能量。

## 7. 验收条件

1. 老师能耗曲线按整车0.1秒采样折算，禁止再次乘16。
2. 单车牵引只产生牵引功率，单车空气紧急制动不产生再生功率。
3. 无受端时允许本车辅助负荷自用，其余再生计为浪费。
4. 电气连通的牵引列车可以吸收制动列车再生能量；拓扑断开后不得跨区吸收。
5. 逐车和系统再生守恒误差均不超过数值容差。
6. 多车结果与输入列表顺序无关。
7. 五车正常场景受电电压大于500V，潮流平衡误差小于1%，P95求解时间满足既有验收。
8. N-1、保护、恢复和一小时稳定性验收不得回退。
