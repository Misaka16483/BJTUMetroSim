## 2. 客流生成器 (PoissonPassengerFlowGenerator)

### 2.1 六时段系数

仿真时刻 `(sim_time_ms / 1000) % 86400` 映射到六时段：

| 时段 | 时间段 | 系数 |
|------|--------|:---:|
| EARLY | 05:00–07:00 | 0.15 |
| AM_PEAK | 07:00–09:00 | 1.45 |
| MIDDAY | 09:00–17:00 | 0.55 |
| PM_PEAK | 17:00–19:00 | 1.15 |
| EVENING | 19:00–22:00 | 0.35 |
| NIGHT | 22:00–24:00 | 0.08 |

0:00–5:00 无运营，系数为 0。

### 2.2 四日型系数

| 日型 | 系数 |
|------|:---:|
| MON_THU（周一至周四，基准） | 1.00 |
| FRI（周五 +8%） | 1.08 |
| SAT（周六 -33%） | 0.67 |
| SUN（周日 -40%） | 0.60 |

### 2.3 到达率公式

```
实际到达率 = base_arrival_rate_pax_per_min × 时段系数 × 日型系数 × line_scale
```

### 2.4 泊松采样

每个 tick 独立采样：
```
λ = 实际到达率 × dt_sec / 60
arrivals = Poisson(λ)  # numpy.random.poisson
```

确定种子 `random_seed=42` 保证可复现。

## 3. 站台状态 (PlatformCrowdState)

| 字段 | 说明 |
|------|------|
| `waiting_pax` | 当前候车人数（进站累积 - 上车消耗） |
| `left_behind_pax` | 滞留人数（= 候车人数） |
| `platform_area_m2` | 站台面积，默认 120m² |
| `platform_density` | density = waiting / area |
| `crowding_level` | LOW (<1.2) / MEDIUM (<2.5) / HIGH (<4.0) / CRITICAL (>=4.0) |

## 4. 停站上下车

### 4.1 一次性模式 (process_train_stop)

```
下车 = onboard_pax × alighting_ratio
上车 = min(候车人数, 剩余容量, 车门通过能力(3pax/s × 30s))
```

停站时间：
```
dwell = 30s + 上车×0.08s + 下车×0.06s + 密度×2s + 调度扣车 + 门故障罚时
       → 限幅 [20s, 90s]
```

### 4.2 连续模式 (exchange_open_door_passengers)

车门 OPEN 期间每 tick 调用一次：
- 下车：按余弦流形 `(1 - cos(π·progress)) / 2` 平滑分布
- 上车：按 `0.3 + 3.0·sin(π·progress)` 形状 + `boarding_credit` 累计控制瞬时速率
- 每次调用后更新 `train.onboard_pax` / `load_factor` / `mass_kg`

## 6. 与仿真引擎的对接

### 6.1 每 tick 调用链

```
_tick()  (engine.py L956-960)
  │
  ├─ 1) station_service.update_arrivals(sim_time_ms, dt_sec)
  │     → 遍历全部 26 个站台，Poisson 生成进站人数
  │     → 存入 _last_arrivals_by_platform
  │
  ├─ 2) _prepare_train_step(train, sim_time_ms)
  │     ├─ DWELLING + door_state==OPEN → _advance_open_door_passengers()
  │     │    → 逐 tick 连续乘客交换
  │     ├─ DWELLING + dwell_remaining_sec≤5.0 → PREPARE_CLOSE
  │     ├─ DWELLING + dwell_remaining_sec==0 → CLOSING (1s过渡)
  │     └─ ARRIVING + transition==0 → _process_station_stop()
  │         → 计划下车总量 + 初始化 30s dwell
  │
  └─ 3) _build_snapshot() 中聚合 totalWaitingPax / maxPlatformDensity
```

### 6.2 生命周期行为

| 操作 | 对客流模块的影响 |
|------|----------------|
| `load()` | 重建全 26 站台（waiting_pax=0） |
| `stop()` | 重建 + `_reset_station_history()` |
| `start()` (IDLE) | 调 `load()` → 重建 |
| `start()` (STOPPED) | 不重建 StationService |

## 8. 已知缺口

| 缺口 | 说明 |
|------|------|
| 终点站反向客流 | GGZ.DOWN / GTG.UP 仍在生成候车人群 |
| 无 OD 矩阵 | 下车比例为固定数值，不追踪起点站 |
| 无换乘 | 单线模型，无跨线客流 |
| 站台无容量上限 | waiting_pax 不封顶 |
| 无闸机/通道延迟 | 乘客瞬间从站外到达站台 |
