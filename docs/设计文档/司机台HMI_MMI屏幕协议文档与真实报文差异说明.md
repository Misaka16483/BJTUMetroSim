# 司机台 HMI/MMI 屏幕协议文档与真实报文差异说明

## 1. 文档目的

本文档记录司机台两块 TCP 屏幕在实物联调中发现的两处协议文档错误，并给出以真实网络报文为依据的正确帧长、字段偏移、程序修正方式和验收方法。

本文档涉及：

- HMI 网络显示屏：TCP 8888 端口。
- MMI 信号显示屏：TCP 9999 端口。
- 上位机向屏幕周期发送的应用层数据帧。

## 2. 结论摘要

| 屏幕 | 端口 | 原文档/原实现 | 真实 TCP 载荷 | 关键错误 |
|---|---:|---:|---:|---|
| HMI 网络显示屏 | 8888 | 572 byte | **570 byte** | 文档多出 2 byte `faultCode`；真实报文中车号位于 568–569 |
| MMI 信号显示屏 | 9999 | 66 byte | **68 byte** | 文档将运行方向/预留与速度字段重叠，导致后续字段整体少 2 byte |

正确的应用层包头为：

```text
8888 HMI: 55 aa 55 aa 3a 02 22 02
          标识字       570   546

9999 MMI: 55 aa 55 aa 3e 00 2a 00
          标识字        62    42
```

> 特别注意：9999 报文的 TCP 实际载荷是 68 byte，但包头 `_uTotalLen` 保留为 62。这是实物程序的历史格式，不能使用该字段作为 TCP 拆包长度。

## 4. 错误一：9999 MMI 信号屏帧长和字段偏移错误

### 4.1 原文档问题

原协议文档声明信号屏总长为 66 byte，同时给出了相互冲突的字段偏移：

- `_nRunDir` 占用偏移 42。
- `_nReserve` 占用偏移 43。
- `_nSpeed` 又被标注为从偏移 42 开始的 4 byte `FLOAT`。

这三个字段不可能同时按原表格存放。原代码为了满足“总长 66 byte”，曾忽略偏移 42–43 的运行方向和预留字段，直接从偏移 42 写入速度。

### 4.2 真实报文证据

参考报文的 IPv4 头包含：

```text
45 00 00 6c ...
```

- IPv4 `Total Length = 0x006c = 108 byte`。
- IPv4 头长 = 20 byte。
- TCP 头长 = 20 byte。
- TCP 实际载荷 = `108 - 20 - 20 = 68 byte`。

应用层起始字节为：

```text
55 aa 55 aa 3e 00 2a 00
```

其中：

- 实际 TCP 载荷：68 byte。
- `_uTotalLen = 0x003e = 62`。
- `_uDataLen = 0x002a = 42`。
