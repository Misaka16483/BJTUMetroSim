# 第2、3模块 AI 优化方法文档

关联模块：

2. 车辆动力学仿真模块

建议项目题目：

```text
基于智能优化的供电约束列车自动停车与节能控制
```

撰写日期：2026-07-07

## 1. 文档目标

本文档用于说明如何在车辆动力学仿真和牵引供电仿真相关问题中引入人工智能方法，形成可落地、可演示、可评估的优化方案。

本文强调一点：AI 不应只是装饰。它应该真正参与以下任务之一：

1. 预测停车误差、能耗、运行时间等结果。
2. 自动搜索控制策略参数。
3. 在扰动场景中选择更稳健的控制策略。
4. 学习列车运行控制策略。
5. 辅助分析不同供电、坡度、质量条件下的运行效果。

## 2. 可引入 AI 的优化问题

前期备选优化问题中，以下问题最适合引入 AI 方法：

| 编号 | 问题名称 | AI 适配度 | 推荐 AI 方法 |
|---|---|---:|---|
| OP-01 | 单列车节能驾驶速度曲线优化 | 高 | 遗传算法、粒子群、贝叶斯优化、强化学习 |
| OP-02 | 停车精度-舒适性-能耗多目标优化 | 很高 | 智能调参、贝叶斯优化、强化学习、回归预测 |

| OP-05 | 欠压/断电扰动下鲁棒运行优化 | 高 | 鲁棒参数搜索、强化学习、分类决策树 |

## 3. AI 方法总览

## 3.1 方法 A：AI 参数寻优

这是最推荐的路线。

基本思想：

```text
人工设计一个规则控制器
  ↓
选出几个关键参数
  ↓
每组参数跑一次仿真
  ↓
根据停车误差、能耗、峰值功率等计算综合评分
  ↓
用智能优化算法自动寻找最优参数
```

适用算法：

```text
遗传算法 Genetic Algorithm, GA
粒子群优化 Particle Swarm Optimization, PSO
贝叶斯优化 Bayesian Optimization, BO
模拟退火 Simulated Annealing, SA
```

优点：

- 不需要真实历史数据。
- 不需要复杂神经网络。
- 与仿真器天然适配。
- 结果容易解释。
- 答辩时能清楚展示优化前后对比。

缺点：

- 需要多次运行仿真。
- 结果质量依赖目标函数设计。

## 3.2 方法 B：预测模型 + 参数筛选

基本思想：

```text
批量运行仿真生成数据集
  ↓
训练模型预测停车误差、能耗、运行时间
  ↓
用预测模型快速筛选候选参数
  ↓
再用真实仿真验证最优候选
```

适用模型：

```text
线性回归
随机森林
XGBoost / LightGBM
MLP 神经网络
高斯过程回归
```

适合预测的输出：

```text
停车误差
总能耗
净能耗
峰值功率
运行时间
最大 jerk
```

优点：

- 有明显 AI 训练过程。
- 能体现数据驱动思想。
- 可以作为“辅助决策模型”。

缺点：

- 需要生成足够训练数据。
- 模型解释性和泛化能力需要验证。
- 若仿真本身很快，预测模型的必要性不如参数寻优直观。

## 3.3 方法 C：强化学习自动控制

基本思想：

```text
把列车运行过程看作一个强化学习环境
智能体根据状态选择牵引/制动/惰行动作
通过奖励函数学习如何停车、节能和避免违规
```

适用算法：

```text
Q-learning
DQN
PPO
SAC
```

优点：

- AI 特征最明显。
- 可以学习非线性控制策略。
- 适合复杂扰动和多目标控制。

缺点：

- 训练不稳定。
- 奖励函数难设计。
- 需要大量训练回合。
- 实训周期内风险较高。

建议：

```text
强化学习可作为扩展方向，不建议作为第一版主线。
```

## 3.4 方法 D：鲁棒优化与蒙特卡洛仿真

基本思想：

```text
随机生成不同质量、坡度、阻力、制动延迟、供电状态
  ↓
同一组控制参数在多场景中运行
  ↓
统计平均表现和最坏表现
  ↓
寻找在多数工况下都表现稳定的参数
```

适用问题：

```text
满载/空载停车误差变化
欠压供电下的控制策略
低黏着或制动延迟下的安全停车
```

可用方法：

```text
随机搜索
遗传算法
贝叶斯优化
分位数优化
最坏场景优化
```

优点：

- 工程意义强。
- 很适合展示“鲁棒性”。
- 不要求复杂 AI 框架。

缺点：

- 仿真次数较多。
- 需要合理设置不确定参数范围。

## 4. 推荐主线方案

最推荐采用：

```text
规则控制器 + 智能优化算法参数寻优
```

推荐题目：

```text
基于智能优化的供电约束列车自动停车与节能控制
```

核心思想：

```text
先设计一个可解释的自动停车规则控制器，
再用 AI 优化算法自动寻找更优控制参数，
使列车在供电约束下停得准、能耗低、峰值功率小、运行平稳。
```

## 5. 推荐 AI 优化建模

## 5.1 控制器结构

建议先使用规则控制器，而不是让 AI 直接输出牵引/制动。

控制逻辑：

```text
if distance_to_target > coast_start_distance:
    牵引或巡航
elif distance_to_target > brake_distance + brake_margin:
    惰行
else:
    制动
```

再根据供电状态调整：

```text
if tractionLimitRatio < 1:
    限制最大牵引级位

if powerStatus == OUTAGE:
    禁止牵引，进入惰行/安全制动
```

## 5.2 待优化参数

第一版建议优化 5 个参数：

| 参数 | 含义 | 建议范围 |
|---|---|---|
| `coast_start_distance` | 距目标点多少米开始惰行 | 100-600 m |
| `brake_margin` | 制动距离安全裕量 | 0-80 m |
| `expected_deceleration` | 预期制动减速度 | 0.3-1.2 m/s² |
| `max_traction_level` | 最大牵引级位 | 1-5 |
| `power_limit_threshold` | 功率限制阈值 | 500-1200 kW |

如果要进一步优化舒适性，可加入：

| 参数 | 含义 | 建议范围 |
|---|---|---|
| `max_brake_level` | 最大制动级位 | 1-5 |
| `level_change_interval` | 级位变化最小间隔 | 0.5-3 s |
| `target_cruise_speed` | 巡航目标速度 | 8-16.7 m/s |

## 5.3 状态变量

AI 优化算法不一定直接看到完整状态，但仿真器需要记录：

```text
positionM
speedMps
accelerationMps2
distanceToTargetM
tractionLevel
brakeLevel
powerKw
energyKwh
tractionLimitRatio
powerStatus
permittedSpeedMps
```

## 5.4 目标函数

推荐综合目标函数：

```text
J =
  w1 * stop_error_norm
  + w2 * energy_norm
  + w3 * peak_power_norm
  + w4 * jerk_norm
  + w5 * time_error_norm
  + w6 * safety_penalty
```

其中：

```text
stop_error_norm = abs(x_stop - x_target) / stop_tolerance
energy_norm = E_net / E_baseline
peak_power_norm = max(P_trac) / P_limit
jerk_norm = max(abs(jerk)) / jerk_limit
time_error_norm = abs(T_arrival - T_plan) / time_tolerance
```

安全惩罚：

```text
safety_penalty =
  1000 * overspeed_count
  + 1000 * power_overload_count
  + 1000 * overrun_count
  + 1000 * emergency_violation_count
```

第一版推荐权重：

```text
w1 = 0.40  停车精度
w2 = 0.20  能耗
w3 = 0.15  峰值功率
w4 = 0.15  平稳性
w5 = 0.10  准点性
w6 = 1.00  安全惩罚
```

如果项目更偏“停车杯”，可改为：

```text
w1 = 0.60
w2 = 0.15
w3 = 0.10
w4 = 0.10
w5 = 0.05
```

## 5.5 约束条件

硬约束：

```text
v(t) <= permittedSpeedMps
P_trac(t) <= P_limit
tractionLimitRatio ∈ [0, 1]
v_stop <= 0.05 m/s
不能越过目标点超过 overrun_limit
供电断电时 tractionLevel = 0
```

软约束：

```text
abs(x_stop - x_target) <= stop_tolerance
abs(T_arrival - T_plan) <= time_tolerance
max(abs(jerk)) <= jerk_limit
max(abs(acceleration)) <= acceleration_limit
```

硬约束违反时直接判为不可行或给极大惩罚。

## 6. 推荐算法路线

## 6.1 第一选择：遗传算法 GA

适合原因：

- 参数范围清楚。
- 不要求目标函数可导。
- 能处理多目标加权。
- 容易实现和解释。

个体编码：

```text
chromosome =
[
  coast_start_distance,
  brake_margin,
  expected_deceleration,
  max_traction_level,
  power_limit_threshold
]
```

适应度函数：

```text
fitness = -J
```

算法流程：

```text
1. 随机生成初始种群
2. 每个个体运行一次完整仿真
3. 根据目标函数计算适应度
4. 选择优秀个体
5. 交叉、变异生成新个体
6. 重复若干代
7. 输出最优参数
```

建议参数：

```text
population_size = 20
generations = 20
crossover_rate = 0.8
mutation_rate = 0.1
```

总仿真次数约：

```text
20 * 20 = 400 次
```

如果仿真很快，这个规模完全可接受。

## 6.2 第二选择：粒子群优化 PSO

适合原因：

- 连续参数优化简单。
- 收敛较快。
- 实现比 GA 更短。

粒子位置：

```text
position =
[
  coast_start_distance,
  brake_margin,
  expected_deceleration,
  max_traction_level,
  power_limit_threshold
]
```

其中 `max_traction_level` 可四舍五入为整数。

算法流程：

```text
1. 初始化粒子位置和速度
2. 每个粒子运行仿真并计算 J
3. 更新个体最优 pbest
4. 更新全局最优 gbest
5. 更新速度和位置
6. 重复迭代
```

建议参数：

```text
particle_count = 20
iterations = 30
w = 0.7
c1 = 1.5
c2 = 1.5
```

## 6.3 第三选择：贝叶斯优化 BO

适合原因：

- 仿真代价较高时更省次数。
- 可以处理少量参数。
- 适合展示“智能搜索”。

基本流程：

```text
1. 随机采样若干组参数
2. 训练代理模型估计 J
3. 用采集函数选择下一组参数
4. 运行仿真得到真实 J
5. 更新代理模型
6. 重复直到预算耗尽
```

建议用于：

```text
参数不超过 6 个
每次仿真较慢
希望用较少仿真次数找到较好解
```

## 6.4 不建议第一版使用：深度强化学习

不建议原因：

- 环境调试成本高。
- 奖励函数不稳定。
- 训练回合多。
- 短期答辩风险较大。

可以作为扩展方向写入：

```text
后续可将列车自动驾驶建模为马尔可夫决策过程，使用 DQN/PPO 学习牵引、惰行和制动策略。
```

## 8.1 MDP 建模

状态：

```text
s_t = [
  distance_to_target,
```

```text
r_t =
  -0.01 * energy_step
  -0.1 * jerk_penalty
```

终止奖励：

```text
if 停车成功:
    r += 100 - 20 * abs(stop_error)
else if 越站:
    r -= 200
else if 超速:
    r -= 200
```

可以围绕以下结论组织：

## 12. 伪代码

## 12.1 仿真评估函数

```python
def evaluate(params, scenario):
    controller = RuleController(
        coast_start_distance=params["coast_start_distance"],
        brake_margin=params["brake_margin"],
        expected_deceleration=params["expected_deceleration"],
        max_traction_level=params["max_traction_level"],
        power_limit_threshold=params["power_limit_threshold"],
    )

    result = run_simulation(controller, scenario)

    stop_error_norm = abs(result.stop_error_m) / scenario.stop_tolerance_m
    energy_norm = result.net_energy_kwh / scenario.baseline_energy_kwh
    peak_power_norm = result.peak_power_kw / scenario.power_limit_kw
    jerk_norm = result.max_abs_jerk / scenario.jerk_limit
    time_error_norm = abs(result.arrival_time_s - scenario.plan_time_s) / scenario.time_tolerance_s

    safety_penalty = (
        1000 * result.overspeed_count
        + 1000 * result.power_overload_count
        + 1000 * result.overrun_count
    )

    score = (
        0.40 * stop_error_norm
        + 0.20 * energy_norm
        + 0.15 * peak_power_norm
        + 0.15 * jerk_norm
        + 0.10 * time_error_norm
        + safety_penalty
    )

    return score, result
```

## 12.2 简单遗传算法流程

```python
population = init_population(size=20)

for generation in range(20):
    scored = []
    for individual in population:
        score, result = evaluate(individual, scenario)
        scored.append((score, individual, result))

    scored.sort(key=lambda x: x[0])
    elites = [item[1] for item in scored[:4]]

    new_population = elites.copy()
    while len(new_population) < 20:
        parent_a, parent_b = select_parents(scored)
        child = crossover(parent_a, parent_b)
        child = mutate(child)
        child = clip_to_bounds(child)
        new_population.append(child)

    population = new_population

best_score, best_params, best_result = min(scored, key=lambda x: x[0])
```

## 13. 风险与规避

| 风险 | 表现 | 规避方式 |
|---|---|---|
| 目标函数设计不合理 | AI 找到奇怪策略 | 使用硬约束和安全惩罚 |
| 优化次数太多 | 运行太慢 | 减小种群、减少迭代、先粗搜后细搜 |
| 模型过于简化 | 结论没有说服力 | 明确“简化模型假设”，加入坡度和供电限制 |
| 强化学习训练失败 | 答辩没有结果 | RL 只做扩展，主线用 GA/PSO |
| 多目标权重争议 | 不同权重结论不同 | 展示不同权重下的策略差异 |
| 参数过多 | 优化不稳定 | 第一版控制在 5 个参数以内 |

## 14. 最终推荐方案

最终建议采用：

```text
规则自动停车控制器 + 遗传算法/粒子群优化 + 供电扰动验证
```

推荐实现优先级：

1. 实现规则控制器。
2. 实现综合评价函数。
3. 实现 GA 或 PSO 参数寻优。
4. 对比人工参数和 AI 优化参数。
5. 加入欠压/断电场景验证鲁棒性。
6. 可选加入随机森林预测模型。

最终答辩主张：

```text
我们将列车自动停车问题建模为供电约束下的多目标优化问题，
使用智能优化算法自动搜索控制参数，
在保证停车精度和安全约束的前提下，
降低能耗、峰值功率和运行冲击。
```
