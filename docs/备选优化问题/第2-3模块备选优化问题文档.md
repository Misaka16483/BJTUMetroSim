# 第2、3模块备选优化问题文档

关联模块：

2. 车辆动力学仿真模块  
3. 牵引供电仿真模块

撰写日期：2026-07-07

## 1. 文档目标

本文档用于从车辆动力学仿真和牵引供电仿真两个模块中发掘有深度、可建模、可仿真、可优化、可答辩的备选优化问题。

这里的“优化问题”不是简单调参数，而是要满足以下条件：

1. 有真实业务背景。
2. 有明确优化目标。
3. 有可量化评价指标。
4. 有车辆动力学和牵引供电之间的耦合关系。
5. 可以在实训周期内做出最小可行版本。
6. 可以继续扩展成更深入的研究问题。

## 2. 领域背景

车辆动力学模块决定列车如何响应牵引、惰行、制动等控制指令，输出速度、位置、加速度、制动距离和停车状态。牵引供电模块决定电压、电流、功率、能耗、再生制动能量和供电故障约束。

两者结合后，最有价值的问题通常不是“让车跑起来”，而是：

- 如何在准点前提下降低能耗？
- 如何在停准前提下降低制动冲击？
- 如何让再生制动能量被更充分利用？
- 如何避免牵引功率峰值过高？
- 供电欠压或断电时如何保证运行安全？
- 如何在车辆性能、信号限速、供电容量之间做权衡？

公开研究和行业工具也体现了这个方向。能效列车控制研究通常关注节能驾驶、坡度、限速、再生制动和运行时间约束；牵引供电仿真标准 EN 50641 明确把列车机械运动计算和牵引供电负荷潮流计算作为核心仿真功能；OpenPowerNet 等工具也强调牵引供电和列车性能协同仿真。

## 3. 优化问题筛选标准

每个候选问题按以下维度评估：

| 维度 | 说明 |
|---|---|
| 业务真实性 | 是否来自真实轨道交通运营、能耗、安全或维护问题 |
| 建模深度 | 是否涉及状态变量、控制变量、约束和目标函数 |
| 两模块耦合 | 是否同时用到车辆动力学和供电仿真 |
| 可实现性 | 是否能在实训周期内做出可演示版本 |
| 可扩展性 | 是否能扩展到论文式分析或更复杂算法 |
| 展示效果 | 是否能用曲线、指标、对比实验讲清楚 |

## 4. 通用建模基础

### 4.1 车辆状态变量

```text
x(t): 列车位置，单位 m
v(t): 列车速度，单位 m/s
a(t): 列车加速度，单位 m/s^2
u_T(t): 牵引级位或牵引力
u_B(t): 制动级位或制动力
P_trac(t): 牵引功率，单位 kW
P_regen(t): 再生制动功率，单位 kW
E(t): 累计能耗，单位 kWh
```

### 4.2 车辆动力学约束

```text
dx/dt = v
dv/dt = (F_trac - F_brake - F_resistance - F_gradient) / m
0 <= v(t) <= v_limit(x)
0 <= F_trac(t) <= F_trac_max(v, power_state)
0 <= F_brake(t) <= F_brake_max(v)
```

### 4.3 供电约束

```text
P_trac(t) <= P_max_section(x)
I(t) = P_trac(t) / U(t)
U_min <= U(t) <= U_nominal
tractionLimitRatio(t) ∈ [0, 1]
```

### 4.4 能耗计算

```text
E_trac = ∫ P_trac(t) dt
E_regen = ∫ P_regen(t) dt
E_net = E_trac - η_reuse * E_regen
```

其中 `η_reuse` 表示再生制动能量利用率。第一版可以取常数，例如 0、0.3、0.6、1.0，用于做敏感性分析。

## 6. OP-01 单列车节能驾驶速度曲线优化

## 6.1 问题背景

真实城市轨道交通中，列车运行需要在满足运行图、限速和停车要求的前提下降低能耗。节能驾驶通常通过合理安排牵引、巡航、惰行和制动阶段实现。研究领域中常称为 energy-efficient train control，也就是节能列车控制。

## 6.2 问题定义

给定线路长度、站点位置、限速、坡度、列车参数和计划运行时间，求一组牵引/惰行/制动控制序列，使列车按时到站并尽量降低净能耗。

## 6.3 决策变量

```text
u_T(t): 牵引级位
u_B(t): 制动级位
mode(t): 运行模式，取值为 TRACTION/CRUISE/COAST/BRAKE
```

离散实现时，可以把全程分成 N 个时间步或位置段：

```text
u_T[k], u_B[k], mode[k], k = 1...N
```

## 6.4 优化目标

单目标版本：

```text
minimize J = E_net
```

带准点惩罚版本：

```text
minimize J = E_net + λ_time * |T_arrival - T_plan|
```

其中：

```text
E_net = E_trac - η_reuse * E_regen
```

## 6.5 约束条件

```text
x(0) = 起点位置
v(0) = 0
x(T) = 目标停车点
v(T) = 0
|T_arrival - T_plan| <= 允许时刻误差
0 <= v(t) <= v_limit(x)
0 <= P_trac(t) <= P_max(x)
牵引和制动不能同时为正
```

## 6.6 分析策略

第一阶段：

1. 构造基准策略：全牵引加速、巡航、固定点制动。
2. 构造节能策略：牵引、惰行、制动三阶段。
3. 通过网格搜索不同惰行起点和制动起点。
4. 对比能耗、运行时间、停车误差。

第二阶段：

1. 使用动态规划搜索离散控制序列。
2. 状态为位置、速度、剩余时间。
3. 动作为牵引、惰行、制动级位。
4. 目标为能耗最小且满足到站时间。

第三阶段：

1. 引入坡度、限速和供电功率限制。
2. 将再生制动利用率作为敏感性参数。
3. 分析不同运行时间裕量对节能效果的影响。

| B | 固定惰行点策略 |

## 7. OP-02 停车精度-舒适性-能耗多目标优化

> 当前实施方案见 [`停车精度-舒适性-能耗多目标优化实验方案V1.md`](../测试与验收/停车精度-舒适性-能耗多目标优化实验方案V1.md)。实施方案结合现有 DCDP、ATO/PID 与低速制动控制，对本节早期设想中的决策变量、目标口径、场景矩阵和验证方法作了收敛与修订；本节保留为选题阶段的原始设计参考。

## 7.1 问题背景

停车任务不是只要停住就行。真实列车进站需要同时考虑停车精度、乘客舒适性、能耗和机械制动磨耗。对于实训中的“停车杯”任务，这个问题特别适合作为核心优化题。

## 7.2 问题定义

给定目标停车点和初始状态，设计制动控制策略，使列车停在目标点附近，同时降低冲击、减少能耗和避免频繁级位切换。

## 7.3 决策变量

```text
brake_start_position: 开始制动位置
brake_level_profile: 制动级位序列
coast_start_position: 惰行开始位置
target_speed_curve: 目标速度曲线
```

## 7.4 优化目标

多目标加权版本：

```text
minimize J =
  w1 * |x_stop - x_target|
  + w2 * max(|jerk|)
  + w3 * E_net
  + w4 * switch_count
```

其中：

```text
jerk = da/dt
switch_count = 牵引/制动级位变化次数
```

也可以设置分层目标：

1. 首先保证停车误差小于 1 m。
2. 在合格停车方案中选择 jerk 最小的。
3. 在舒适性接近时选择能耗最低的。

## 7.5 约束条件

```text
v_stop <= 0.05 m/s
|x_stop - x_target| <= 停车容差
0 <= brake_level <= max_brake_level
max(|a|) <= 舒适性上限
max(|jerk|) <= 冲击上限
不触发紧急制动
```

## 7.6 分析策略

第一阶段：

1. 设计固定制动点策略。
2. 扫描不同制动起点和制动级位。
3. 得到停车误差热力图。

第二阶段：

1. 设计目标速度曲线。
2. 用 PID 或规则控制跟踪目标速度。
3. 调整 PID 参数或规则阈值。

第三阶段：

1. 把停车误差、jerk、能耗做多目标加权。
2. 改变权重，生成 Pareto 候选策略。
3. 解释不同权重下策略取舍。

## 7.7 仿真实验设计

关键图表：

- 制动起点-制动级位-停车误差热力图
- 速度-距离曲线
- 加速度-时间曲线
- jerk-时间曲线

## 7.8 适合实训的最小版本

推荐实现：

```text
基于制动距离预测的自动停车控制
```

核心逻辑：

```text
estimated_brake_distance = v^2 / (2 * expected_deceleration)
if distance_to_target <= estimated_brake_distance + margin:
    apply_brake
else:
    coast_or_traction
```

然后把 `expected_deceleration` 和 `margin` 作为优化参数。

## 8. OP-03 再生制动能量最大化利用优化

## 8.1 问题背景

电力列车制动时可以把动能转换为电能。真实系统中，再生能量可能被附近牵引列车吸收、被储能设备吸收、反馈电网，或因无法吸收而被制动电阻消耗。因此，仅仅“产生再生能量”不等于“有效利用再生能量”。

## 8.2 问题定义

在保证停车安全和准点的前提下，优化制动过程或牵引过程，使再生制动能量产生更多、可利用比例更高，从而降低净能耗。

## 8.3 决策变量

单列车版本：

```text
brake_start_position
brake_level_profile
regen_priority_ratio
mechanical_brake_ratio
```

双列车扩展版本：

```text
departure_time_offset
traction_window_of_adjacent_train
braking_window_of_current_train
```

## 8.4 优化目标

单列车简化版本：

```text
maximize E_regen
```

更真实版本：

```text
maximize E_reused = min(E_regen, E_absorbable)
```

或：

```text
minimize E_net = E_trac - E_reused
```

## 8.5 约束条件

```text
停车误差 <= 容差
制动距离 <= 可用距离
再生制动力 <= 再生制动能力上限
机械制动力 + 再生制动力满足总制动需求
供电电压不能超过允许上限
```

## 8.6 分析策略

第一阶段：

1. 只统计理论再生能量。
2. 比较不同制动强度、不同制动起点下再生能量。
3. 分析“早轻制动”和“晚强制动”的差异。

第二阶段：

1. 设置再生能量利用率 `η_reuse`。
2. 比较 `η_reuse = 0, 0.3, 0.6, 1.0` 下净能耗变化。

第三阶段：

1. 引入相邻列车或虚拟负载。
2. 只有当其他列车牵引功率需求与当前列车再生功率时间重叠时，再生能量才可被吸收。
3. 优化发车间隔或牵引窗口，提高重叠面积。

## 8.7 适合实训的最小版本

推荐实现：

```text
单列车制动策略对再生能量的影响分析
```

输出：

- 不同制动策略下理论再生能量
- 不同再生利用率下净能耗
- 停车误差是否合格

如果时间充裕，再扩展成“两列车牵引-制动重叠优化”。

## 9. OP-04 牵引功率峰值削减优化

## 9.1 问题背景

牵引供电系统不仅关心总能耗，也关心峰值功率和最大电流。过高的牵引峰值会增加供电系统压力，可能引起电压下降、设备过载或容量需求上升。真实运营中，削峰填谷和限制牵引峰值是重要能源管理问题。

## 9.2 问题定义

在满足运行时间、限速和停车要求的前提下，优化牵引级位和速度曲线，使最大牵引功率尽量低，同时能耗不过度增加。

## 9.3 决策变量

```text
traction_level_profile
max_allowed_traction_level
acceleration_limit
coast_start_position
```

## 9.4 优化目标

```text
minimize J = w1 * max(P_trac(t)) + w2 * E_net + w3 * |T_arrival - T_plan|
```

## 9.5 约束条件

```text
P_trac(t) <= P_section_limit
I(t) <= I_section_limit
T_arrival <= T_plan + tolerance
v(t) <= v_limit(x)
x(T) = x_target
v(T) = 0
```

## 9.6 分析策略

1. 设计基准策略：高牵引快速加速。
2. 设计削峰策略：限制最大牵引级位或限制加速度。
3. 比较峰值功率、总能耗和运行时间。
4. 分析不同计划运行时间裕量对削峰效果的影响。

## 9.7 适合实训的最小版本

实现三个策略：

```text
策略 A：最大牵引级位 5
策略 B：最大牵引级位 3
策略 C：动态限制牵引级位，使功率不超过阈值
```

对比：

```text
最大功率
总能耗
运行时间
停车误差
```

这个问题展示很直观，适合用功率曲线和柱状图讲清楚。

## 10. OP-05 欠压/断电扰动下鲁棒运行优化

## 10.1 问题背景

供电系统可能出现欠压、限流、区段断电等异常。真实运营中，列车控制系统必须在异常供电条件下保证安全，尽可能完成运行或安全停车。

## 10.2 问题定义

给定供电扰动场景，设计控制策略，使列车在牵引能力受限或断电时仍能满足安全约束，并尽量降低延误和停车误差。

## 10.3 决策变量

```text
degraded_mode_policy
traction_limit_response
coasting_policy
safe_braking_policy
recovery_policy
```

## 10.4 优化目标

```text
minimize J =
  w1 * safety_violation_count
  + w2 * delay
  + w3 * |x_stop - x_target_or_safe_point|
  + w4 * passenger_discomfort
```

安全优先时，目标可分层：

1. 不违反安全约束。
2. 不越过禁止位置或目标安全停车点。
3. 尽量减少延误。
4. 尽量减少冲击。

## 10.5 约束条件

```text
供电断电时 F_trac = 0
欠压时 F_trac <= F_trac_max * tractionLimitRatio
v(t) <= v_limit(x)
必要时触发制动或惰行
不能越过安全停车边界
```

## 10.6 分析策略

1. 构造供电扰动场景：欠压、限流、断电、恢复。
2. 分析基准策略在扰动下的失败模式。
3. 设计降级策略：
   - 欠压：降低牵引级位，延长加速时间。
   - 断电：惰行并准备制动。
   - 恢复：平滑恢复牵引，避免功率突增。
4. 比较扰动前后运行时间、停车误差、最大 jerk 和安全事件。

## 10.7 适合实训的最小版本

推荐实现：

```text
供电断电触发安全停车策略
```

场景：

```text
列车运行至 600m
供电区段断电 20s
控制器切换为惰行/制动
若无法到站则在安全点前停车
供电恢复后继续运行或结束场景
```

这个问题工程味强，也容易体现“真实业务场景”和“异常处理能力”。

## 11. OP-06 制动混合策略优化

## 11.1 问题背景

列车制动通常涉及再生制动和机械制动。再生制动有利于节能，但受速度、电机能力、供电吸收能力限制；机械制动可靠但会带来磨耗和热负荷。因此需要优化制动力分配。

## 11.2 问题定义

在满足总制动力需求和停车精度的前提下，尽可能多使用再生制动，减少机械制动磨耗，同时避免因再生制动力不足导致停车失败。

## 11.3 决策变量

```text
F_regen(t)
F_mech(t)
regen_priority_ratio(t)
brake_level_profile(t)
```

## 11.4 优化目标

```text
minimize J =
  w1 * E_mech_brake_loss
  - w2 * E_regen
  + w3 * |x_stop - x_target|
  + w4 * max(|jerk|)
```

也可写成：

```text
maximize E_regen - α * mechanical_brake_usage
```

## 11.5 约束条件

```text
F_regen + F_mech = F_required_brake
0 <= F_regen <= F_regen_max(v, power_state)
0 <= F_mech <= F_mech_max
停车误差 <= 容差
```

## 11.6 分析策略

1. 设置再生制动上限随速度变化。
2. 优先使用再生制动，不足部分由机械制动补足。
3. 比较纯机械制动、固定比例混合制动、再生优先制动。
4. 输出再生能量、机械制动使用量、停车误差。

## 11.7 适合实训的最小版本

可以不实现复杂电机模型，只设定：

```text
F_regen_max = k * v
```

速度越低，再生制动力越小。这样能直观体现低速阶段需要机械制动补足。

## 12. OP-07 坡度线路上的能耗-时间权衡优化

## 12.1 问题背景

线路坡度会显著影响列车能耗。上坡需要更多牵引能量，下坡可以惰行或制动。真实节能驾驶会利用坡度信息安排牵引和惰行。

## 12.2 问题定义

给定含上坡、下坡和平坡的线路，优化牵引、惰行和制动位置，使列车在满足运行时间的前提下尽量降低能耗。

## 12.3 决策变量

```text
traction_start/end_position
coast_start/end_position
brake_start_position
traction_level_profile
```

## 12.4 优化目标

```text
minimize J = E_net + λ * |T_arrival - T_plan|
```

## 12.5 约束条件

```text
v(t) <= v_limit(x)
上坡速度不能低于最小运行速度
下坡不能超速
到站停车误差 <= 容差
```

## 12.6 分析策略

1. 构造三种线路：平坡、先上后下、先下后上。
2. 使用同一控制策略运行，观察能耗差异。
3. 针对坡度优化惰行点。
4. 比较坡度感知策略与坡度无关策略。

## 12.7 适合实训的最小版本

实现一个坡度数组：

```text
0-500m: 平坡
500-1000m: 上坡 10‰
1000-1500m: 下坡 -8‰
```

然后比较：

```text
不考虑坡度的控制策略
考虑坡度提前牵引/提前惰行的策略
```

## 13. OP-08 供电区段容量约束下的自动驾驶优化

## 13.1 问题背景

在供电区段容量有限时，车辆控制不能只追求最快运行。若牵引功率超过区段容量，可能造成过载或电压下降。

## 13.2 问题定义

给定供电区段最大功率和电流限制，优化列车牵引控制，使列车满足运行任务且不超过供电容量。

## 13.3 决策变量

```text
traction_level_profile
acceleration_limit
section_power_limit_response
```

## 13.4 优化目标

```text
minimize J =
  w1 * E_net
  + w2 * |T_arrival - T_plan|
  + w3 * overload_penalty
```

其中：

```text
overload_penalty = Σ max(0, P_trac(t) - P_limit(x))^2
```

## 13.5 约束条件

```text
P_trac(t) <= P_limit(section)
I(t) <= I_limit(section)
tractionLimitRatio 根据供电模块实时更新
```

## 13.6 分析策略

1. 给不同区段配置不同功率上限。
2. 基准策略不考虑供电限制，可能触发过载。
3. 优化策略根据区段容量调整牵引级位。
4. 对比过载次数、运行时间和能耗。

## 13.7 适合实训的最小版本

实现“功率限幅控制”：

```text
if predicted_power > section_power_limit:
    reduce traction level
```

这容易与供电模块联动，也容易验收。

## 14. OP-09 列车参数不确定下的停车控制优化

## 14.1 问题背景

真实列车运行中，载客量、轮轨状态、制动响应和阻力参数会变化。控制策略如果只针对固定参数调优，可能在质量变化或低黏着条件下停车误差变大。

## 14.2 问题定义

在列车质量、阻力和制动响应不确定的情况下，设计停车控制策略，使多种工况下停车误差都保持在允许范围内。

## 14.3 决策变量

```text
brake_margin
expected_deceleration
PID 参数
安全冗余参数
```

## 14.4 优化目标

```text
minimize J =
  E_scenario[|x_stop - x_target|]
  + λ * max_scenario(|x_stop - x_target|)
  + μ * passenger_discomfort
```

也可设置鲁棒目标：

```text
minimize max_scenario(|x_stop - x_target|)
```

## 14.5 不确定参数

```text
massKg: 标称值 ± 20%
resistanceCoefficient: 标称值 ± 30%
brakeDelayMs: 0-800ms
availableAdhesion: 正常/低黏着
```

## 14.6 分析策略

1. 建立多个工况场景。
2. 用同一控制参数分别仿真。
3. 统计平均停车误差和最坏停车误差。
4. 通过网格搜索寻找鲁棒参数。

## 14.7 适合实训的最小版本

推荐实现：

```text
质量变化下的自动停车鲁棒性分析
```

比较：

```text
空载
标称载荷
满载
```

输出每种工况停车误差。

## 15. OP-10 能耗预测与控制策略自适应优化

## 15.1 问题背景

当系统积累多次仿真数据后，可以根据历史运行数据预测能耗、停车误差或制动距离，并调整控制参数。这个方向更偏人工智能应用。

## 15.2 问题定义

利用历史仿真数据训练一个预测模型，预测给定初始状态和控制参数下的停车误差或能耗，然后选择更优控制参数。

## 15.3 决策变量

```text
coast_start_position
brake_start_position
brake_margin
expected_deceleration
```

## 15.4 优化目标

```text
minimize predicted_cost =
  w1 * predicted_stop_error
  + w2 * predicted_energy
  + w3 * predicted_time_error
```

## 15.5 分析策略

1. 批量仿真生成数据集。
2. 特征包括初始速度、距离目标点、坡度、质量、供电状态、控制参数。
3. 标签包括停车误差、能耗、运行时间。
4. 训练回归模型。
5. 用模型筛选候选控制参数，再用仿真验证。

## 15.6 可选模型

```text
线性回归
决策树
随机森林
梯度提升树
简单神经网络
```

## 15.7 适合实训的最小版本

不建议作为第一选择，因为它依赖数据量和调参时间。但可作为加分方向：

```text
用批量仿真数据训练停车误差预测模型
```

展示：

- 预测停车误差 vs 实际停车误差
- 预测辅助选出的控制参数是否优于人工参数

## 16. 推荐选题组合

## 16.1 最推荐主线：OP-02 + OP-04 + OP-05

这是最适合实训项目的组合：

```text
OP-02 停车精度-舒适性-能耗多目标优化
OP-04 牵引功率峰值削减优化
OP-05 欠压/断电扰动下鲁棒运行优化
```

理由：

1. 与“停车杯”和自动停车场景高度相关。
2. 能同时体现车辆动力学和供电约束。
3. 能做出清楚的 Demo。
4. 指标容易解释：停车误差、能耗、峰值功率、扰动响应。
5. 工程复杂度可控。

推荐项目标题：

```text
供电约束下的列车自动停车与节能控制优化
```

## 16.2 节能研究主线：OP-01 + OP-03 + OP-07

适合把项目包装成“绿色轨道交通节能运行优化”。

核心故事：

```text
通过优化速度曲线、惰行点和制动策略，降低牵引能耗，并提高再生制动能量利用潜力。
```

优点：

- 研究味强。
- 图表好看。
- 能与绿色交通、双碳目标联系。

风险：

- 若车辆模型太简化，节能结论可能显得薄。
- 需要设计较好的基准策略。

## 16.3 供电安全主线：OP-04 + OP-05 + OP-08

适合把项目包装成“牵引供电约束下的安全运行仿真”。

核心故事：

```text
列车自动驾驶不能只看速度和停车点，还必须考虑供电区段容量、欠压、断电和功率峰值。
```

优点：

- 与牵引供电模块结合紧密。
- 工程场景真实。
- 异常场景展示效果强。

风险：

- 需要供电模块至少能输出功率、电压、状态和 tractionLimitRatio。

## 17. 推荐最终选题

综合考虑实训周期、展示效果、技术深度和模块耦合程度，最推荐选择：

```text
题目：供电约束下的列车自动停车与节能控制优化
```

## 17.1 核心问题

在给定线路、目标停车点、车辆参数和牵引供电状态的情况下，如何生成牵引、惰行、制动控制策略，使列车：

1. 停得准。
2. 运行平稳。
3. 能耗较低。
4. 牵引功率峰值不过高。
5. 在供电欠压或断电扰动下仍能安全降级。

## 17.2 总目标函数

```text
minimize J =
  w1 * |x_stop - x_target|
  + w2 * E_net
  + w3 * max(P_trac)
  + w4 * max(|jerk|)
  + w5 * delay
  + w6 * safety_penalty
```

其中：

```text
safety_penalty =
  1000 * overspeed_count
  + 1000 * emergency_violation_count
  + 1000 * power_overload_count
```

## 17.3 约束条件

```text
v(t) <= v_limit(x)
P_trac(t) <= P_limit(x)
tractionLimitRatio(t) ∈ [0, 1]
|x_stop - x_target| <= stop_tolerance
v_stop <= 0.05 m/s
max(|a|) <= acceleration_limit
max(|jerk|) <= jerk_limit
```

## 17.4 分析策略

第一步：建立基准控制器。

```text
根据距离目标点和当前速度做规则控制：
距离远时牵引
进入惰行区间时惰行
进入制动距离时制动
```

第二步：选取可调参数。

```text
coast_start_position
brake_margin
expected_deceleration
max_traction_level
power_limit_threshold
```

第三步：进行参数搜索。

```text
网格搜索或随机搜索
每组参数跑完整仿真
记录停车误差、能耗、峰值功率、jerk、运行时间
筛选满足约束的最优策略
```

第四步：加入供电扰动。

```text
正常供电
欠压供电
区段断电
供电恢复
```

第五步：对比策略。

```text
基准策略
停车优先策略
节能优先策略
供电约束策略
鲁棒策略
```

## 17.5 最小可行 Demo

Demo 场景：

```text
线路长度：1500 m
目标停车点：1500 m
初始速度：0
限速：16.7 m/s
供电标称电压：1500 V
最大供电功率：1200 kW
故障场景：800 m 处欠压或断电
```

展示内容：

- 列车速度曲线
- 位置曲线
- 牵引/制动级位曲线
- 功率曲线
- 能耗统计
- 停车误差
- 正常供电 vs 欠压/断电对比

## 17.6 预期结论

可能得到的结论包括：

1. 适当提前惰行可降低能耗，但可能增加运行时间。
2. 过晚制动可以缩短运行时间，但停车误差和 jerk 风险更高。
3. 限制牵引峰值会降低供电压力，但可能增加运行时间。
4. 欠压场景下，降级控制能避免过载和过度牵引。
5. 断电场景下，安全停车策略比继续追求到站更合理。
