### 距离2.0目标的判断

当前已经具备“可演示的单车/供电/客流局部闭环”和多车、人工驾驶的开发中界面，但距离2.0完成定义仍有明显差距。按Stage退出条件判断：

| Stage | 状态 | 尚未满足的退出条件 |
|---|:---:|---|
| Stage 0 数据与公共契约 | 进行中 | 上下行权威Link、老师车辆52点曲线、车头/车尾、生命周期、稳定多车快照和统一契约尚未完成验证 |
| Stage 1 多级位动力学与宏观客流 | 进行中 | 正式多级位、客流质量反馈、双向场景、3车一致展示和人数守恒回归未通过 |
| Stage 2 联锁、调度与生命周期 | 尚未形成闭环 | CI未进主tick，无车库发收车、固定闭塞MA/ATP和冲突进路端到端验收 |
| Stage 3 多车供电完整闭环 | 局部领先、未退出 | 单车/教学供电基础较强，但再生匹配、5车稳定运行、能量平衡和N-1全链路当前回归失败 |
| Stage 4 时延、回放与研究实验 | 基本未开始 | 无通信时延实验、基线对比、权威回放和八场景一键报告 |

近期优先级应是：①先恢复全量测试和前端构建；②冻结Stage 0数据/状态契约；③完成3车“客流→质量→动力学”闭环；④再把CI/MA/ATP接入主tick；⑤最后完成多车再生、N-1和八场景验收。三维视景和工程级MVB/CAN不应挤占当前2.0核心闭环资源。

> 依据《任务2：需求描述》定义，按 7 个子系统划分。
> 每个系统按 **前端 → 后端 → 前后端对接** 三部分展开，标注 ✅ 已实现 / ⚠️ 部分实现 / ❌ 未实现。
