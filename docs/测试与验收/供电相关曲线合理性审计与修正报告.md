# 供电相关曲线合理性审计与修正报告

## 1. 审计范围与结论

本轮审计覆盖牵引供电页面的全部趋势量、后端字段来源、车辆控制到供电潮流的闭环、数据采样语义，以及 1 倍速/10 倍速显示一致性。线路和供电参数均按脱敏数据构造的“平行世界 9 号线”工程估计使用，不宣称复现真实北京地铁 9 号线。

审计发现此前曲线中同时存在三类问题：真实控制脉冲、潮流/限幅口径不闭合、前端快照混叠。现已完成 P0 修正。修正后的关键结论如下：

- 接近限速时，DCDP 不再因车辆模型先钳制速度而把 100% 牵引误判成合法巡航；运行时的轨迹前馈会随速度误差连续淡出。
- 低压限牵会同步回代列车交付功率、网络电流和实际牵引力，不再只缩小 API 中的限幅比例。
- 再生限幅以实际接受功率比例为唯一依据，实际电制动力、累计再生能量和分账保持一致；无受端时不再用 1000 V 作为哨兵值。
- 后端保留每个 0.25 s 物理微步的曲线样本，前端不再把 10 倍速下收到的稀疏 WebSocket 帧当成等间隔物理采样。
- 接触轨同时给出左右端空间电流、净区段注入和平均穿越电流，避免对称双边供电显示为零负荷。
- 报告能耗改为每个供电区段最后一条累计值之和，不再对逐 tick 累计量再次求和。

## 2. 曲线字段与正确口径

| 页面曲线/指标 | 后端权威字段 | 单位与符号 | 审计后的解释 |
|---|---|---|---|
| 最低列车电压 | `curveSamples[].minVoltageV` | V | 当前微步所有受电列车的最低端电压；无列车时为 `null`，不伪造 750 V |
| 最大牵引所电流负载率 | `maxSubstationLoadRatio` | p.u. | 牵引所电流绝对值/额定电流；允许超过 1.0 |
| 牵引所净功率 | `netSubstationPowerKw` | kW | 直流母线净功率；正值流入牵引网，负值表示净回馈 |
| 整流机组直流侧输出 | `rectifierPowerKw` / `rectifierDcBusOutputKw` | kW | 端电压乘整流电流；不能称为交流输入 |
| 等效直流源功率 | `equivalentDcSourcePowerKw` | kW | 空载电压乘整流电流 |
| 牵引所内部损耗 | `substationInternalLossKw` | kW | 等效直流源功率减直流母线输出，满足精确分解 |
| 网络与设备电阻损耗 | `networkLossesKw` | kW | 当前降阶模型中的站内、馈线/轨网及再生传输电阻损耗；不包含储能内部待机损耗 |
| 候选再生功率 | `generatedRegenKw` | kW | 制动指令和电机能力给出的可用再生潜力，不等于最终实际电制动能量 |
| 本车辅助自用 | `selfConsumedRegenKw` | kW | 再生功率在本车辅助负荷内直接使用 |
| 跨车吸收 | `absorbedRegenKw` | kW | 经直流牵引网送达其他牵引列车的功率 |
| 储能充/放电 | `storageChargeKw` / `storageDischargeKw` | kW | 超级电容直流母线侧功率 |
| 变电所回馈 | `feedbackRegenKw` | kW | 回馈装置直流侧吸收；供电图中按负方向绘制 |
| 未回收再生潜力 | `wastedRegenKw` | kW | 未被网络、储能或回馈装置接受的候选功率；不直接宣称已由制动电阻耗散 |
| 再生传输损耗 | `regenTransferLossesKw` | kW | 再生源到受端路径的电阻损耗 |

再生分账在每个微步满足：

```text
候选再生功率
= 本车辅助自用
 + 跨车/储能/回馈路径源端接受功率
 + 未回收再生潜力
```

每条传输路径满足：

```text
路径源端接受功率 = 路径受端送达功率 + 路径电阻损耗
```

## 3. 已修正的问题

### 3.1 限速前牵引脉冲

原离线规划先调用会把速度钳制到车辆最高速度的动力学模型，再判断是否超速。因此车辆即使在限速附近持续 100% 牵引，得到的下一步速度仍等于上限，错误候选不会被淘汰。运行时又把 `MAX_TRACTION` 直接解释为 100% 前馈，形成截图中的牵引功率脉冲。

修正内容：

- 在速度钳制前计算原始加速度、原始下一步速度和位置，并先按线路/车辆限速淘汰候选。
- 巡航候选按 Davis 运行阻力和坡度计算动态维持力，下坡时可生成巡航制动候选。
- 运行时读取规划点的实际牵引百分比，并在 PID 死区 0.06 m/s 到完整前馈误差 0.35 m/s 之间连续加权。
- 速度曲线缓存版本从 2 升至 3，旧的不合理规划不会被复用。

1423 m、22.22 m/s 代表性单车复算中，规划点不存在“距上限 0.06 m/s 内仍为满牵引”的状态；CRUISE 段运行指令约 10.85% 平滑上升到 18.11%，没有 0%/100% 点刹式跳变。全程按期停车于 1423.073 m，最终速度为 0。

### 3.2 低压限牵不回代潮流

原实现先按完整恒功率请求求出低至 500 V 的电压和电流，之后才缩小 `tractionLimitRatio`，导致列车实际牵引力/API 交付功率已经降低，但牵引所和馈线曲线仍按完整请求取流。

修正后以 650 V 为当前模型的可持续恒功率下限；能力不足时，求解器直接削减整流侧交付功率和电流，并按“辅助负荷优先、剩余功率供牵引”得到实际牵引交付值。

5 列车同区重载回归结果：

- 最低列车电压：650.0 V；
- 每车牵引接受比例：约 0.8476；
- 列车端 `U × I` 合计与“牵引交付 + 辅助负荷”合计一致；
- 全网功率平衡误差比例约 0.0041%，低于 1% 验收阈值。

### 3.3 再生接受、过压限幅和实际制动力不一致

原实现先分配全部再生功率，之后又按源端电压二次缩小 `regenLimitRatio`，引擎据此降低实际电制动力，但已接受功率和累计量不重算。无受端时还会人为填入 1000 V。

修正内容：

- 按 `(源列车, 源馈线)` 累计再生电流，并限制源端电压不超过 900 V；
- 以“本车自用 + 网侧源端接受”除以候选再生功率得到唯一 `regenLimitRatio`；
- 未被接受的部分进入“未回收再生潜力”，实际制动由气制动补足；
- 无外部接受路径时显示可达牵引网开路电压，而不是 1000 V 哨兵值。

### 3.4 接触轨和牵引所功率口径

接触轨旧字段使用 `左端原始电流 - 右端原始电流`，既会让对称双边供电显示为 0，又会让纯穿越潮流放大一倍。现定义：

```text
左端空间电流 = 左端原始电流
右端空间电流 = -右端原始电流
净区段注入 = 左端原始电流 + 右端原始电流
平均穿越电流 = (左端原始电流 - 右端原始电流) / 2
```

旧 `currentA` 为兼容保留，但明确等于平均穿越电流。保护负载率仍按两端最大绝对电流计算。

牵引所同时发布直流母线输出、内部损耗和等效直流源功率，并逐站满足：

```text
等效直流源功率 = 整流机组直流侧输出 + 牵引所内部损耗
```

## 4. 采样与显示修正

- 后端曲线窗口覆盖最近 180 s，每个 0.25 s 物理微步一条，样本带 `simTimeMs`、`electricalSolveTimeMs` 和 `samplePeriodMs`。
- 1 倍速和 10 倍速均执行相同物理微步；自动化回归确认推进相同仿真时长时，两者曲线时间戳和采样周期完全一致。
- WebSocket 只发布最新状态，序列前跳视为正常合帧，不再拒绝新帧并触发 REST 重同步风暴。
- 最低电压无列车时显示空缺；SVG 曲线对 `null` 断线，不再画成额定电压。
- 电压轴至少覆盖 650–900 V，并随实测值扩展；负载轴至少覆盖 0–1.05 p.u.，过载不会被裁掉。
- “功率与能量”拆为“直流侧供电功率”和“再生功率分配”，所有曲线均为 kW；横轴标注仿真秒。

## 5. 验收与回归

本轮新增或强化的验收覆盖：

- 低压削减后的网络电流、列车端功率、交付功率和实际牵引力一致；
- 再生接受比例、实际电制动力和逐车累计量一致；
- 无受端再生不出现 1000 V 假值，源端不超过 900 V；
- 接触轨左右端、净注入和平均穿越电流定义一致；
- 每个牵引所的直流源功率分解闭合；
- 求解失败同时回滚超级电容和牵引所累计能量；
- 1 倍速/10 倍速物理微步曲线一致；
- 报告能耗只取各供电区段末值；
- 前端 TypeScript 生产构建和静态检查通过。

验证结果：在只检出本次提交、不包含工作区其他修改的独立工作树中，供电、报告、引擎、ATO 和速度曲线定向回归合计 79 项通过；前端 `npm run build` 通过，`npm run lint` 无错误（仓库原有警告保留）。全仓库 `unittest discover` 在 240 s 执行窗口内未结束且未输出提前失败，因此不将超时误报为全量通过。

## 6. 保留限制与后续建议

以下内容不影响本轮曲线合理性验收，但应作为后续模型深化项，而不是在当前联合优化实验中同时改动：

1. 当前是降阶准稳态直流潮流，不包含按每个列车抽头离散的完整节点电压/轨段电流剖面；现有接触轨字段是可守恒的区段降阶量。
2. 650 V 采用硬削减边界。若后续有受流器/变流器实测曲线，可升级为 500–650 V 平滑降额函数。
3. 页面回馈功率是回馈装置直流侧吸收；若要报告交流侧输入/输出，需要显式加入整流和逆变效率，不能只改标签。
4. 超级电容阈值策略仍是硬触发，可能形成真实折线尖角。建议在供电闭环基线稳定后单独加入迟滞、功率斜率和 SOC 目标带，并重新做联合优化对照。
5. `wastedKw` 作为兼容字段暂时保留；新实验和答辩统一称“未回收再生潜力”，避免把候选能量误称为已发生的制动电阻耗散。
