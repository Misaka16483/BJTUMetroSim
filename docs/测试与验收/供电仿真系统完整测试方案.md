# 供电仿真系统完整测试方案

## 1. 目标

本方案验证牵引供电仿真是否具备支撑后续参数优化、运行图优化、再生能量利用优化和储能定容选址的基础。验证分为三个层次：

1. **软件正确性**：算法、单位、符号、状态机、数据契约无逻辑错误。
2. **工程合理性**：结果满足直流牵引供电的物理守恒、单调关系和合理范围。
3. **真实线路标定度**：评估参数证据是否足以代表北京地铁9号线。当前质量等级为 `ENGINEERING_ESTIMATE`，该层不作“实线精确”结论。

## 2. 测试对象和边界

测试对象包括：

- DC750V牵引所、馈电臂、接触轨、回流轨和联络开关拓扑；
- 恒功率列车负荷的多电源准静态直流潮流；
- 车辆动力学输出的牵引功率、再生功率和辅助负荷；
- 再生能量的本车自用、跨车吸收、路径损耗、交流回馈、超级电容充电和浪费；
- 牵引所停运、馈电臂断开、接触轨停电、联络开关越区供电和恢复；
- 超级电容 SOC、充放电功率、效率、削峰阈值和宿主牵引所联动；
- 引擎同步、SQLite记录、API快照和供电页面。

不在当前精度验收范围内：电磁暂态、PWM、谐波、保护配合定值、杂散电流、温升和实线 PSCADA 绝对误差。

## 3. 基线和可追溯性

| 项目 | 基线 |

| 模型版本 | `LINE9-DC750-V1.0` |

| 馈电臂 | 36条 |

## 4. 分级和失败处置

| 级别 | 含义 | 处置 |
|---|---|---|
| P0 | 守恒破坏、NaN/Inf、越界、故障设备仍供电、错误快照发布 | 禁止优化实验，必须修复 |
| P1 | 单调关系反向、结果依赖输入顺序、故障恢复不可重复 | 必须修复 |
| P2 | 性能超预算、显示与API不一致、记录不完整 | 本轮修复或明确限制 |
| P3 | 真实线路参数证据不足 | 保留 `ENGINEERING_ESTIMATE`，列入校准计划 |

## 5. 量化验收矩阵

### 5.1 拓扑、参数和量纲

| ID | 测试 | 验收标准 |
|---|---|---|
| TOP-01 | 设备数量、ID、外键和里程顺序 | 全部唯一；引用存在；无自环和负长度 |
| TOP-02 | 上下行、左右供电范围 | 每一正线公里标有且仅有预期的正常供电路径 |
| TOP-03 | 严格拓扑加载 | 缺少设备、来源或参数时显式失败，不静默补齐 |
| TOP-04 | 参数范围 | 电压、电阻、容量、效率、SOC均有限且符合配置约束 |
| DIM-01 | 功率维度 | `P(kW)=U(V)*I(A)/1000`，相对误差 < 1% |
| DIM-02 | 线路损耗维度 | 损耗非负；解析算例与 `I^2R` 相对误差 < 1% |
| DIM-03 | 符号约定 | 整流供电为正，交流回馈在图上为负，内部台账不以符号代替去向 |

### 5.2 解析基准与求解器

| ID | 测试 | 验收标准 |
|---|---|---|
| SOL-01 | 无负荷 | 整流电流、线损和列车功率为0；无 NaN/Inf |
| SOL-02 | 单电源恒功率闭式解 | 电压绝对误差 < 0.5V，电流相对误差 < 0.1% |
| SOL-03 | 对称双边供电 | 两端电流差/平均值 < 1% |
| SOL-04 | 距离单调性 | 其他参数不变，远离主供电源时电压不上升 |
| SOL-05 | 负荷单调性 | 牵引功率增加时电流和损耗不减、电压不升 |
| SOL-06 | 电阻敏感性 | 0.8/1.0/1.2倍电阻下电压递减、损耗递增 |
| SOL-07 | 输入顺序不变性 | 列车列表排列变化后逐车结果误差 < 1e-6 |
| SOL-08 | 收敛与平衡 | 正常和验收压力工况全收敛；平衡误差目标 < 0.1%，硬上限 < 1% |
| SOL-09 | 不可解/不收敛 | 不发布坏快照，保留上一有效值并暂停引擎 |

### 5.3 再生能量与储能

| ID | 测试 | 验收标准 |
|---|---|---|
| REG-01 | 系统再生守恒 | 生成=本车自用+跨车吸收+储能充电+回馈+路损+浪费；误差 < 1e-6kW 或 1e-9相对值 |
| REG-02 | 逐车再生守恒 | `available=accepted+wasted`，误差 < 1e-6kW |
| REG-03 | 分配优先级 | 本车辅助→可达列车→储能→能馈→浪费 |
| REG-04 | 电气隔离 | 断开供电路径后不得跨区吸收或回馈 |
| REG-05 | 再生限制 | 无受端时出现浪费与 `regenLimitRatio<1`，制动力由空气制动补足 |
| ESS-01 | SOC边界 | 始终位于[minSoc,maxSoc]，越界容差 < 1e-9 |
| ESS-02 | 额定功率 | 单设备汇总充/放电功率不超额定值，不同时充放电 |
| ESS-03 | 能量和效率 | SOC变化与充放电、效率、待机损耗的积分一致，误差 < 1e-6kWh |
| ESS-04 | 削峰阈值 | 低于1000kW可达负荷不放电；高于阈值时仅供应超出部分 |
| ESS-05 | 故障联动 | 宿主牵引所停运时储能退出，复位后按配置恢复 |

### 5.4 故障、拓扑切换和保护

| ID | 测试 | 验收标准 |

## 6. 执行阶段

1. **T0 基线冻结**：记录代码、配置、环境和已知限制。
2. **T1 快速门禁**：编译、拓扑、解析基准、守恒、故障、储能和API专项。
3. **T2 属性与敏感性**：随机工况、负荷/距离/电阻单调性、步长和输入顺序。
4. **T3 综合闭环**：车辆、客流、联锁、供电、储能和记录同步运行。
5. **T4 压力与长时**：10/20/40车和一小时求解。
6. **T5 前端验收**：真实浏览器中检查曲线、切页、故障和恢复。
7. **T6 全量回归**：运行全部 Python 测试和前端生产构建。

## 7. 优化研究就绪门槛

仅当以下条件同时满足，才标记为“可用于教学型优化研究”：

1. P0/P1 缺陷为0，全部硬性守恒、边界和故障测试通过。
2. 解析基准、定性单调性、确定性和一小时稳定性通过。
3. 优化所需指标可追溯至原始负荷、拓扑状态和参数版本。
4. 无参数的变化不得被误解为算法改进；所有对比实验固定随机种子和输入。
5. 结论中明确“工程估算拓扑”，不将保护、真实所址和设备容量作为实线结论。
6. `POWER-CREDIBILITY-BASELINE-V1` 四项准入门全部通过；点号按 1 基展示、RTDB 行参数按 0 基传输，映射契约指纹固定。
