# 闭环时序与储能筛选实验 V1

## 1. 实验目的

本实验承接《运行图多车储能控制实验 V1》，将固定轨迹储能回放升级为候选驱动的闭环筛选：每一组发车、牵引和制动时序候选都必须重新运行主仿真，形成不同的列车运动与受流轨迹，然后在该轨迹上优化超级电容控制。

实验对象仍是脱敏数据构造的“平行世界 9 号线”工程估计模型，不代表真实北京地铁 9 号线，也不用于设备定值或节能承诺。

## 2. 变量与实现语义

| 变量 | 实际作用 | 正值语义 |
|---|---|---|
| `departureSpreadSec` | 加到运行图各时段发车间隔 | 扩大发车间隔 |
| `tractionTimingSec` | 沿 DCDP 速度曲线移动牵引段查询位置 | 延后退出牵引 |
| `brakeTimingSec` | 沿 DCDP 速度曲线移动制动段查询位置 | 延后进入制动 |
| `storageChargeLimitKw` | 超级电容充电功率上限 | 增大可充功率 |
| `storageDischargeLimitKw` | 超级电容放电功率上限 | 增大可放功率 |
| `storageTriggerKw` | 超级电容放电触发功率 | 提高放电门槛 |

时序偏置根据列车当前速度换算为 DCDP 曲线上的位置偏移。制动段优先级高于牵引段，避免牵引偏置遮蔽提前制动请求。所有候选仍经过低速制动防抖、联锁、运行图偏差、动力学闭合、供电潮流和末端 SOC 门禁。

## 3. 两层多保真方案

直接把完整主仿真嵌入大种群 NSGA-II 的成本过高，因此 V1 采用“主仿真时序筛选 + 轨迹内储能优化”的嵌套方案。

第一层以 0.5 秒主仿真步长、2 个车底和 60 秒固定窗口筛选三组时序：

| 工况 | 发车扩展 | 牵引偏置 | 制动偏置 |
|---|---:|---:|---:|
| `BASELINE` | 0 s | 0 s | 0 s |
| `EARLY_COAST_BRAKE` | 0 s | -1 s | -1 s |
| `STAGGERED_EARLY_COAST_BRAKE` | 10 s | -1 s | -1 s |

第二层对每条候选轨迹运行一次种子为 `20260714`、种群 6、进化 1 代的储能 NSGA-II。筛选出的推荐时序以 0.25 秒主仿真步长重新采集轨迹，并固定筛选阶段储能参数进行 1.0/0.5 秒电气步长复验，不重新挑选参数。

复现命令：

```powershell
python tools/run_closed_loop_joint_experiment.py `
  --max-duties 2 `
  --capture-seconds 60 `
  --screen-tick-seconds 0.5 `
  --validation-tick-seconds 0.25 `
  --evaluation-step-seconds 1.0 `
  --storage-seed 20260714 `
  --population 6 `
  --generations 1 `
  --wall-timeout-sec 600 `
  --artifact-dir outputs/closed-loop-joint-screening-v1 `
  --output outputs/closed-loop-joint-screening-v1.json
```

## 4. 筛选结果

三组时序轨迹的帧指纹均不同，证明候选确实改变了主仿真轨迹，而非只修改供电回放标签。三组工况均满足 SOC、控制质量和运行偏差约束。

相对于全局零吞吐基线：

| 工况 | 储能后净交流侧能耗 | 汇总交流峰值 | 弃再生比例 | 综合结论 |
|---|---:|---:|---:|---|
| `BASELINE` | 21.4179 kWh | 3029.5124 kW | 0.801947 | 推荐 |
| `EARLY_COAST_BRAKE` | 23.9855 kWh | 3511.2645 kW | 0.803756 | 能耗 +7.90%，峰值 +15.90% |
| `STAGGERED_EARLY_COAST_BRAKE` | 21.7305 kWh | 3029.6685 kW | 0.801771 | 能耗 +2.25%，峰值基本不变 |

提前退出牵引并提前制动没有节能，反而触发了更大的后续补偿牵引；其最大运行时分偏差从基线 34.925 秒增至 46.425 秒，最大出发偏差从 35.925 秒增至 47.425 秒。增加 10 秒发车间隔缓解了峰值，但不足以抵消该控制偏置的能耗代价。

因此本轮推荐保留原始发车、牵引和制动时序，仅采用筛选得到的储能参数：

| 参数 | 推荐值 |
|---|---:|
| 充电功率上限 | 1754.663518 kW |
| 放电功率上限 | 373.734287 kW |
| 放电触发功率 | 131.611620 kW |

筛选窗口内，相对全局零吞吐基线，推荐方案净交流侧能耗下降 3.6545%，弃再生比例下降 8.9953%，峰值不变；末端 SOC 偏差为 3.6111%。

## 5. 0.25 秒高保真复验

固定上述时序和储能参数后，0.25 秒主仿真复验通过：

- 固定储能参数可行，1.0 秒电气评价下末端 SOC 偏差为 4.0840%；
- 0.5 秒电气半步复验仍可行，末端 SOC 偏差为 4.0702%；
- 相对同一高保真轨迹的无储能工况，净交流侧能耗从 22.8818 kWh 降至 22.0452 kWh；
- 弃再生比例从 0.876419 降至 0.788083；
- 汇总交流峰值保持 3058.0196 kW；
- 低速快速制动再施加为 0，牵引与制动重叠样本为 0。

## 6. 结论与下一轮变量

本轮得到的是有价值的阴性时序结果：对全部列车统一施加 -1 秒牵引与制动提前量不是合适的节能方向。当前运行图时序应保持不变，储能控制可继续使用推荐值作为下一轮中心点。

下一轮不再扩大相同方向的统一偏置，而应采用更细的局部变量：

1. 将牵引退出与制动进入拆开做 `{-0.5, 0, +0.5}` 秒局部因子试验，识别导致补偿牵引的具体变量；
2. 只对高功率重叠明显的车次或区间施加偏置，而不是全线统一修改；
3. 在 4 车、240 秒窗口复验局部推荐项，并加入多起始时刻；
4. 若局部筛选仍无收益，应停止时序调整，把研究重点转向储能触发值、储能位置和容量敏感性。
