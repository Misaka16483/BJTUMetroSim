# 供电AI优化实验报告

## 1. 实验配置

| 项目 | 配置 |
|---|---|
| 模型 | 9号线DC750V牵引供电V1 |
| 数据质量 | `ENGINEERING_ESTIMATE` |
| 算法 | 确定性种子的进化搜索 |
| 种群规模 | 8 |
| 进化代数 | 3 |
| 随机种子 | 20260711 |
| 每候选列车数 | 12 |
| 每候选时段数 | 16 |
| 单时段长度 | 5 s |

执行命令：

```powershell
python tools/run_power_optimization.py --population 8 --generations 3 --seed 20260711
```

原始结果保存于 `outputs/power-optimization-report.json`，试验及候选保存于 `outputs/power_experiments.sqlite`。

## 2. 结果摘要

| 问题 | 基线得分 | 最佳得分 | 加权目标改善 | 最佳候选 |
|---|---:|---:|---:|---|
| 再生匹配 | 6207.140 | 43.394 | 99.30% | 发车错峰14.1818 s；再生相位偏移1.4066 s |
| 牵引错峰 | 23138.856 | 7130.249 | 69.18% | 发车错峰29.2856 s |
| 回馈容量 | 6173.603 | 3994.236 | 35.30% | 三个候选能馈所各2000 kW |
| N-1鲁棒运行图 | 23285.783 | 5643.192 | 75.77% | 发车错峰29.2621 s；牵引强度系数0.8342 |

四个最佳候选均满足：潮流全部收敛、功率平衡误差小于1%、最低列车电压不低于500 V。

## 3. 解释

1. 再生匹配改善较大，是因为同步基线中牵引与制动相位重叠较少，错峰后再生能量可被邻近牵引列车吸收。
2. 牵引错峰降低了同时启动造成的整流峰值和限牵罚项，但引入运行图偏差。
3. 回馈容量最优值达到2000 kW搜索上界，说明当前容量成本权重下继续增容仍有边际收益；后续应使用真实设备报价、寿命和电价重新标定成本项，并扩大搜索范围确认拐点。
4. N-1方案降低牵引强度并错峰发车，以运行时间代价换取变电所停运时的电压和限牵鲁棒性。

## 4. 验收结论

- 四类问题均可通过统一API和CLI批量执行。
- 同请求、同算法、同种子的最佳候选和得分一致。
- 每个候选保留决策变量、目标分量、硬约束、原始潮流指标和代数信息。
- HTTP创建/查询、SQLite持久化和四问题优化由 `tests/test_power_experiments.py` 覆盖。

上述改善比例是加权目标函数的相对变化，不是真实线路节能率、投资收益率或运行图改进率。真实运营结论必须在拓扑、设备容量、能耗曲线、运行图和成本参数校准后重新计算。
