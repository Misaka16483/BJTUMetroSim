## 2. 测试目标与通过准则

目标是验证最终成品能够被安装、构建、启动和操作，并对每个主要功能域至少完成一层自动化验证，对安全或状态型功能完成跨模块验证。

通过准则：

1. Python 源码可编译，完整 unittest 无失败或错误。
2. 仿真启动、暂停、单步、恢复、停止和重启状态一致，停止后运行态正确清理。
3. 多车、进路、移动授权、道岔/区段占用、追踪和折返满足联锁约束。
4. 客流守恒，自动泊松与人工输入模式可切换，载客量能传递至质量和能耗。
5. 供电潮流收敛，故障、开关、再生制动、储能、N-1、记录回放和性能门槛通过。
6. API 主要读写路径在真实 HTTP 服务中返回 2xx 且业务结果不为 `ok=false`。
7. 前端 lint 不得有 error，TypeScript 与生产构建必须成功，入口和模块资源可被服务。
8. 无真实设备或外部密钥的功能，必须至少通过协议、序列化、模拟器或本机回环测试，并记录实物验收缺口。

## 6. 本轮发现并修复的问题

1. 远端合并后客流服务初始化缺少 `StationFlowConfig` 相关定义，导致完整回归出现 113 个级联错误。已恢复基于版本化客流 profile 的初始化，并保留新加入的泊松开关和需求倍率。
2. `App.tsx` 存在合并残留：重复 `modeIndex`、重复驾驶页按钮和未闭合/重复容器，导致 TypeScript/Vite 无法解析。已清理 JSX 结构，生产构建恢复。
3. 多个长仿真测试在紧循环中使用异步 DCDP，仿真时钟远快于墙钟，产生与机器调度相关的假失败。量化/进路/车门测试改用确定性 ATO；DCDP 异步能力仍由专门测试覆盖。
4. 部分进路断言仍按旧的“列车头停在站台段起点”和一次性区段释放语义编写。已按站台居中停车、车尾逐段释放和物理折返语义更新。
5. 两个独立 audit 工具仍使用旧生命周期/原地折返约定。已更新为“stop 清空运行编组”和“先安排有进路保护的物理折返”。
6. 量化供电脚本同样受异步 DCDP 墙钟影响，已加入可重复的确定性 ATO 模式并重新通过全部门槛。
7. 远端 `e79c212` 修复顶栏启动控件不可见及仿真状态返回问题。已保留远端 `App.tsx`/API 实现，并补充源码契约测试和真实 Vite 代理生命周期验收，防止按钮再次被放入隐藏容器或调用链断裂。
8. 原自动发车面板只能查看和调整已有任务，不能新增完整交路。本轮新增后端队列 API、前端表单和 Store 调用链，并补齐编号/间隔/状态校验、失败原子性、即时持久化和停止重建测试。
9. 运行图编辑与仿真 tick、恢复运行并发时存在状态竞争风险；供电拓扑操作又可能形成反向锁序。本轮统一运行图编辑边界和 `tick → power` 锁序，并在停止线程超时后显式报错，避免静默清理仍在运行的状态。

## 7. 非阻断项与人工验收建议

1. 前端项目当前未配置 Playwright、Cypress 或同类真实浏览器测试依赖，因此本轮未做 DOM 级逐页点击和截图比对。作为补偿，已执行 6 项源码契约测试、生产构建、HTTP 资源检查，并通过 Vite 代理真实执行生命周期与自动发车写操作；仍需在目标浏览器人工检查尺寸适配、图层交互、弹窗和中文显示。
2. 当前环境未安装 `websockets`，后端明确降级为 HTTP 轮询；WebSocket 推送未做进程级验收。若交付要求实时推送，应安装项目约定版本后补做断线重连和高频推送测试。
3. PLC、司机台、网络屏、信号屏和视景机没有真实硬件。协议、状态机和 UDP 回环均通过，但上线前仍需按真实 IP/端口完成连通、断线恢复、字节序和长稳测试。
4. 高德地图接口依赖外部网络和有效密钥，本轮不把外部服务可用性作为本地发布门槛。
5. lint 的 17 条 warning 主要是 React hook 依赖、Fast Refresh 导出方式、可疑空表达式和数值精度提示；不阻断构建，但建议在后续维护版本清零。
6. 生产 JS 主包约 1.92 MB（gzip 520.63 kB），Vite 提示超过 500 kB。功能不受影响，后续可按功能页动态 import 做代码分包。
7. 唯一跳过项为需要环境变量 `ENGINE_TRAJECTORY_JSONL` 的真实联锁/运行图轨迹文件验证；公共引擎快照适配测试已通过。具备现场轨迹后应补跑该项。
8. 前端当前将 90 秒最小发车间隔作为常量，与后端配置保持一致但存在未来漂移风险；后续宜由运行图 API 返回权威 `minimumHeadwayS`。
9. 若极端情况下停止线程超过 5 秒触发 `SIMULATION_STOP_TIMEOUT`，应等待旧线程退出并重试 stop 完成清理，不应直接 start。正常停止路径和本轮验收均未触发该异常。
