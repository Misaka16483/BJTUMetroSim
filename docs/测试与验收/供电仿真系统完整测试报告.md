# 供电仿真系统完整测试报告

## 1. 结论

测试日期：2026-07-13。

当前 9 号线 DC750V 牵引供电仿真已通过软件正确性、物理一致性、数值稳定性、系统闭环、数据契约和前端显示验证，具备开展教学与科研性质优化问题的基础条件。自动验收 13 项判据全部通过，全仓 302 项 Python 回归测试全部通过，前端 TypeScript 构建和 lint 通过。

本结论限定为 `ENGINEERING_ESTIMATE` 工程估算模型可用于相对比较、方案筛选和算法研究，不代表真实北京地铁 9 号线的工程标定结果或保护定值认证。

## 2. 测试对象

- 10 座牵引变电所、36 个馈电臂、18 个接触轨分段、18 个回流轨分段、9 个联络开关。
- 双边供电、联络供电、变电所和馈线故障、接触轨失电及保护动作。
- 多列车恒功率潮流、受电弓位置和跨供电分区取流。
- 再生能量本车自用、跨车吸收、超级电容、交流回馈、传输损耗和制动电阻浪费。
- 超级电容 SOC、充放电功率、效率、待机损耗和故障退出。
- 车辆动力学、客流载荷、ATO/手动控制与供电限制闭环。
- SQLite 记录、命令回放、实验接口、API 响应和供电专用前端。

## 3. 验证方法

采用四层证据链：

1. 解析基准：单电源恒功率直流电路闭式解、对称双边供电电流平分、空载零功率。
2. 性质测试：负载和距离单调性、量纲一致性、能量守恒、SOC 边界、输入顺序无关、固定随机种子多车不变量。
3. 场景验收：N-1、联络开关、再生分配、储能循环、客流质量传播、多车和一小时连续仿真。
4. 系统验收：引擎、API、SQLite、实验接口、前端构建和真实浏览器交互。

详细测试矩阵见《供电仿真系统完整测试方案》；机器可读结果见 `outputs/power-validation-report.json`，自动摘要见 `outputs/power-validation-report.md`。

## 5. 再生与储能核验

- 再生功率按本车辅机自用、跨车吸收、储能充电、交流回馈、传输损耗和浪费完整分账。
- 每条再生路径满足 `generated = delivered + losses`，网络汇总守恒误差为零至浮点精度。
- 超级电容直接吸收优先于交流回馈，跨车吸收优先于储能充电，符合当前策略定义。
- 充放电不同时发生；SOC 始终位于 0.2 至 0.9；主变电所停运时储能退出运行。
- 10 秒过程分别采用 10 秒、1 秒和 0.1 秒步长积分，最终储能量在浮点精度内一致。
- 无吸收端或吸收能力不足时，再生浪费和再生限幅告警会出现，不再把全部电制动能量误记为可利用能量。

## 6. 本轮发现并修复的问题

### 6.1 故障恢复拓扑漂移

原实现执行大双边 N-1 时会闭合相邻联络开关，但 `SUBSTATION_RESTORE` 仅恢复变电所和馈线，联络开关可能永久保持闭合，使恢复后的网络不等于故障前网络。

修复后，网络保存每次停运前的受影响馈线和联络开关状态，恢复时精确还原；若相邻变电所仍停运，则保留必要联络状态。新增“精确恢复”和连续 5 次故障恢复无漂移测试，均通过。

### 6.2 验收脚本版本漂移

原脚本仍按旧步长固定运行 360 tick，客流列车无法在限定 tick 内到达 1000 m；再生守恒也未计入本车自用和储能充电。现已改为按仿真秒数运行，并使用完整再生分账口径。

### 6.3 前端历史与最低电压

供电页切换到其他页面后曾卸载并清空趋势历史；最低电压计算也可能把非实际列车电压的默认值计入。现已保持供电视图挂载，并按有效列车受电数据计算最低电压。

浏览器实测中，趋势样本从 7 点切换到宏观页后增长至 162 点，返回供电页未重置；指标卡最低电压与列车受电表最小值一致，控制台无错误。

## 7. 数据与接口验证

- API 供电拓扑、仿真状态和供电优化实验接口通过专项测试。
- SQLite 已记录列车电压、变电所功率、超级电容、再生能量路径、求解状态和供电命令。
- 故障及开关命令可延时执行、记录、导出和回放。
- 求解不收敛或平衡误差超限时，不发布坏快照；引擎暂停并暴露失败原因，储能状态回滚。
- 前端 `npm run build` 与 `npm run lint` 均成功。lint 存在仓库既有警告，但本轮没有供电相关错误。

## 8. 优化研究就绪性

以下问题已经具备重复实验条件：

- 多车发车时序与再生能量跨车利用优化。
- 超级电容位置、容量、功率、SOC 阈值和削峰阈值优化。
- N-1 工况下联络开关重构和限牵策略优化。
- 以网侧购电、峰值负荷、最低电压、线路损耗、再生浪费和运行延误为多目标的联合优化。
- 客流载荷、车辆质量、ATO 曲线与供电约束联合优化。

每个优化实验必须固定拓扑版本、场景、随机种子和时间步长，保存原始 trial、约束违例和基线结果。不得只报告最优值而不报告可行率、重复性和相对基线改善率。

## 9. 剩余限制

- 10 座牵引所位置、内阻、馈线和接触轨电阻主要为工程估算，不是运营方实测参数。
- 当前为准静态 DC 潮流，不模拟毫秒级电磁暂态、保护继电器动作波形、杂散电流和谐波。
- AC 侧等值为容量受限整流/回馈模型，未建立完整中压交流网和三相潮流。
- 供电保护为教学规则，不可用于真实保护整定。
- 老师提供的实时数据库示例目前仅可读，且记录号 1000 至 1470 的量纲和点表语义尚未完整确认，不能直接作为校准真值。

因此，当前系统适合研究“在同一模型假设下哪个方案更优”，但在获得真实图纸、SCADA 或实测数据前，不应宣称绝对电压、能耗或设备容量与真实 9 号线完全一致。
