# 运行图多车储能控制实验 V1

## 1. 实验定位

本实验用于验证“平行世界 9 号线”工程估计模型中，固定运行图多车轨迹与超级电容控制的可比优化流程，不宣称复现真实北京地铁 9 号线效果。

本阶段使用主仿真系统生成的固定 4 车轨迹，优化超级电容充电功率上限、放电功率上限和放电触发功率。由于静态轨迹回放不能改变列车运动，牵引/制动时序变量在本阶段固定为零；本报告不把储能回放结果表述为已经完成了运行图时序与储能的闭环联合优化。

## 2. 修正项

正式实验前完成了两项门禁修正：

1. ATO 在终端制动区、低速且尚未进入蠕行释放时，禁止小制动力在单个 0.25 秒控制周期内直接跨零，消除了轨迹中的短暂释闸后再施加现象，同时保留原 18%/s 正常制动释放斜率。
2. 储能优化器支持实验级决策变量边界和基线。4 车短窗口不再沿用 12 车代理实验的 3400–3800 kW 放电阈值，而是在 0–2500 kW 内搜索，并把末端 SOC 偏差不超过 5% 作为硬约束。

## 3. 可复现实验配置

```powershell
python tools/run_timetable_power_experiment.py `
  --max-duties 4 `
  --capture-seconds 240 `
  --tick-seconds 0.25 `
  --evaluation-step-seconds 1.0 `
  --seeds 20260714,20260717,20260719 `
  --population 12 `
  --generations 3 `
  --wall-timeout-sec 600 `
  --trajectory outputs/timetable-power-trajectory-post-pid-v2.jsonl `
  --output outputs/timetable-power-experiment-post-pid-v2.json
```

轨迹共 961 帧、3844 个列车样本，车组固定；覆盖 2510 个运动车样本、1715 个牵引样本、592 个制动样本和 1047 个再生样本。供电回放采用 1 秒评价步长和每步 2 次电气子步，推荐解另以 0.5 秒评价步长复验。

## 4. 正式结果

全部执行门禁与假设检查通过：

| 指标 | 零吞吐基线 | 推荐解 | 相对改善 |
|---|---:|---:|---:|
| 净交流侧网能耗 | 155.0619 kWh | 148.7342 kWh | 4.0808% |
| 汇总交流侧峰值 | 7314.2979 kW | 5685.9969 kW | 22.2619% |
| 弃再生比例 | 0.706897 | 0.606379 | 14.2196% |

推荐储能控制参数为：

| 变量 | 推荐值 |
|---|---:|
| 充电功率上限 | 1714.649469 kW |
| 放电功率上限 | 1881.035434 kW |
| 放电触发功率 | 705.090318 kW |

储能初始能量为 10.0000 kWh，窗口末为 9.5918 kWh，末端 SOC 偏差为 2.0409%；窗口内累计充电 6.9008 kWh、放电 6.4594 kWh。0.5 秒半步复验的末端 SOC 偏差为 2.1980%，解仍可行。

控制与运行门禁结果：

- 低速制动状态切换 8 次，2 秒内快速再施加 0 次；
- 牵引与制动同时出现的样本 0 个；
- 4/4 车次进入运行，无卡车；
- 速度超限 RMSE 为 0.1896 m/s，最大速度 21.48 m/s；
- 最大出发偏差 77.851 秒，最大区间运行时分偏差 77.351 秒，均在本次运行图 120 秒验收容差内；
- 基线、无储能基线、三个种子推荐解和半步长复验均可行。

三个 NSGA-II 重复共完成 140 次不重复评估，可行率 54.29%。推荐效用的改善范围为 9.48%–13.52%，中位数为 12.55%。等评估预算随机搜索的中位改善为 13.13%，NSGA-II 的最好结果为 13.52%，随机搜索最好结果为 13.14%；因此本轮只能说明 NSGA-II 找到了有效解，不能宣称其在小规模三变量问题上全面优于随机搜索。

## 5. 结论边界与下一步

本实验说明：在同一脱敏线路、运行图、供电拓扑和工程估计参数下，满足末端 SOC 可比性与控制质量门禁的超级电容策略可以同时降低本窗口的网侧能耗、峰值和弃再生比例。

仍需继续完成：

1. 将时序变量接入可重新运行主仿真的轨迹提供器，使每个候选都真实改变发车、牵引和制动轨迹，再开展闭环联合优化；
2. 在不同起始时刻、客流强度和更长运行窗口复验，避免把单个 240 秒窗口推广为全天效果；
3. 使用更多评估预算或针对三变量问题的网格/贝叶斯基线，进一步判断 NSGA-II 的算法收益；
4. 保持 `ENGINEERING_ESTIMATE` 质量标识，不将结果用于真实设备定值、保护整定或真实线路节能承诺。
