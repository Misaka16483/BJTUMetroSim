# 模块7：调度与运行图仿真模块 — 需求调研

---

## 一、模块边界

|问题|回答|
|---|---|
|该模块模拟什么现实对象|地铁列车在多站线路上的运行图及实际执行过程|
|该模块输入什么|线路站序、区间运行时分、停站时分、发车间隔、客流到达率、最小追踪间隔|
|该模块输出什么|每列车每站计划到发时刻、实际到发时刻、准点率、乘客等待时间、满载率、延误链路|
|该模块影响哪些其他模块|接收：模块4信号与列控仿真模块提供间隔约束、模块6车站与站台仿真模块提供停站时分；输出：模块2车辆动力学仿真模块提供运行计划、模块3牵引供电仿真模块提供功率需求|
|本项目最小实现范围|单线多站多列车，分时段固定发车间隔的运行图，含乘客候车队列与KPI统计|

### 场景清单

|场景|描述|预期结果|
|---|---|---|
|平峰正常运营|8站单线，5列车，5分钟等间隔，运行2小时|到发偏差小于10秒，满载率30%至50%|
|早高峰加密|6:30至9:00间隔压缩到2分钟，客流3倍于平峰|满载率低于120%，等待时间低于3分钟|
|单点延误传播|早高峰某列车在C站车门故障延误120秒|后3至5车连带延误，延误递减，20分钟内恢复|
|跳站运营|1列快车跳过3个低客流站|快车全程缩短15%至25%，被跳站等待不超过2个间隔|
|临时限速|某区间限速25km/h持续30分钟|经过列车区间时间延长，准点率暂时下降后恢复|

### 评估指标清单

|指标|公式|合格线|
|---|---|---|
|准点率|偏差绝对值小于120秒的车次数量除以总车次数|大于95%|
|平均等待时间|每客等待秒数总和除以总乘客数|高峰小于180秒，平峰小于360秒|
|区间满载率|当前载客量除以定员|高峰低于120%|
|追踪间隔合规率|间隔不低于最小追踪间隔的采样数除以总采样数|等于100%|
|延误恢复时间|准点率回到90%的时刻减去首次延误时刻|低于20分钟|
