# 轨道交通综合仿真系统软件设计文档

版本：v0.1  
日期：2026-07-07  
依据文档：`docs/需求与调研文档/轨道交通仿真系统需求说明书.md`  
设计范围：Phase 0、Phase 1 详细设计；Phase 2、Phase 3 概要设计  

## 1. 文档目的

本文档用于将已审核的需求说明书转化为可实施的软件设计方案。当前版本重点解决：

1. 如何解析老师提供的真实线路数据。
2. 如何建立统一数据模型、仿真时钟、消息总线和记录体系。
3. 如何完成 Phase 0 的数据基线与工程框架。
4. 如何完成 Phase 1 的单车运行、ATO 控制、基本列控和 ATP 防护。
5. 如何为 Phase 2 的联锁/车站/供电/客流闭环预留扩展接口。
6. 如何为 Phase 3 的 AI 优化与调度深化预留实验框架。

本文档不是最终编码规范，但应作为后续详细开发、接口定义、任务拆分和测试验收的共同基线。

## 2. 设计原则

1. 真实数据优先：第一版应从 `线路数据(1).xls` 构建电子地图，避免手写玩具线路。
2. 分阶段可运行：每个阶段都应形成可演示闭环，而不是等全部模块完成才运行。
3. 安全规则兜底：AI 或 ATO 输出不得绕过 ATP、限速、MA、紧急制动等安全约束。
4. 模块低耦合：业务模块通过统一消息和状态快照交互，减少直接互相调用。
5. 可回放可复现：所有仿真输入、状态、指令、事件和指标应可记录和复盘。
6. 可替换实现：UI、算法、通信协议可替换，核心数据模型和接口保持稳定。

## 4.2 运行时数据流

Phase 1 单车运行时，每个 tick 的推荐数据流如下：

```text
SimulationClock 产生 tick
  ↓
M8 EnvironmentService 注入到期扰动
  ↓
M1 TrackMapService 查询列车当前位置线路条件
  ↓
M4 TrainControlService 计算允许速度、MA、ATP 请求
  ↓
M5 CabControlService 生成 ControlCommand
  ↓
M3 PowerService 生成 PowerState 或默认牵引限制
  ↓
M2 VehicleDynamicsService 更新 TrainState
  ↓
M6 StationService 判断停车/到站事件
  ↓
M11 Recorder 记录状态、指令、事件、指标
  ↓
M12 UI 刷新显示
```

注意：Phase 1 的供电模块可先使用默认 `tractionLimitRatio=1.0`，但接口必须保留。

## 7. Phase 0 详细设计

## 7.1 线路数据解析器

### 7.1.1 职责

`LineDataImporter` 负责读取 `线路数据(1).xls`，解析 33 个工作表中的核心表，并生成统一电子地图缓存。

### 7.1.2 输入

```text
docs/线路数据(1).xls
```

### 7.1.3 输出

```text
data/cache/line_map.json
data/cache/line_map.sqlite
data/cache/import_report.json
```

第一版至少输出 JSON；SQLite 可作为 Phase 0 后半段或 Phase 1 优化。

### 7.1.4 解析规则

| 规则 | 说明 |
|---|---|
| 元信息 | 第 1-4 行为元信息，真实数据从第 5 行开始 |
| 行列范围 | 优先使用第 2 行声明行数/列数 |
| 空引用 | `65535` 转为 `null` |
| 长度 | cm 转为 m |
| 速度 | cm/s 转为 m/s 和 km/h |
| 里程 | `K0+313.000` 转为 313.000 m |
| 十六进制 | `0x55`、`0xaa` 等保留原始值，暂不强行解释 |

### 7.1.5 核心解析表

Phase 0 必须解析：

```text

道岔表
信号机表
车站表
站台表
```

Phase 2 再充分使用：


### 7.1.6 引用校验

导入时执行以下检查：

| 检查 | 处理 |
|---|---|
| Seg 起终点引用点/道岔不存在 | ERROR |
| Seg 相邻 Seg 不存在 | ERROR |
| 信号机引用 Seg 不存在 | ERROR |
| 站台引用 Seg 不存在 | ERROR |
| 限速引用 Seg 不存在 | ERROR |
| 坡度引用 Seg 不存在 | ERROR |
| 进路引用信号机不存在 | ERROR |
| 进路引用计轴区段不存在 | ERROR |
| 对象偏移量超出 Seg 长度 | WARN 或 ERROR，视字段确认情况 |
| 未解释的属性编码 | INFO |

导入器不应因 INFO 中断；ERROR 默认中断场景加载，除非使用 `allowInvalidMap=true`。

## 7.2 电子地图服务

### 7.2.1 组件

| 组件 | 职责 |
|---|---|
| TrackGraph | 保存 Seg 拓扑图 |
| TrackMapRepository | 从 JSON/SQLite 加载基础对象 |
| TrackQueryService | 对外提供位置、限速、坡度、站台、信号查询 |
| MapValidationService | 数据一致性检查 |

### 7.2.2 核心接口

```python
class TrackQueryService:
    def get_segment(seg_id: int) -> Segment: ...
    def get_next_segments(seg_id: int, direction: str) -> list[int]: ...
    def locate_train(position: TrackPosition) -> TrackLocation: ...
    def get_speed_limit(seg_id: int, offset_m: float) -> SpeedRestriction | None: ...
    def get_gradient(seg_id: int, offset_m: float) -> Gradient | None: ...
    def get_nearest_platform(seg_id: int, offset_m: float, direction: str) -> Platform | None: ...
    def get_next_signal(seg_id: int, offset_m: float, direction: str) -> Signal | None: ...
```

### 7.2.3 位置表达

Phase 0/1 推荐同时维护两种位置：

```text
拓扑位置：segId + offsetM
累计运行位置：positionM
```

原因：

1. `segId + offsetM` 适合查线路数据。
2. `positionM` 适合车辆动力学和曲线绘制。
3. 二者可由 `TrackNavigator` 做映射。

### 7.2.4 TrackNavigator

`TrackNavigator` 负责沿 Seg 拓扑推进列车位置。

输入：

```text
current segId
current offsetM
travelDistanceM
direction
switchStateMap
```

输出：

```text
new segId
new offsetM
passedSegments
endOfTrack flag
```

第一版策略：

```text
优先走 endForwardSegId
遇到道岔时按默认定位方向走
侧向/反位由 Phase 2 联锁道岔状态控制
```

### 7.3.3 发布订阅规则

1. 高频状态只保留最新快照，并由 Recorder 按采样频率记录。
2. 事件不得覆盖，应全部记录。
3. 命令每个 tick 最多采用一个最终控制命令。
4. ATP 覆盖命令优先级高于 ATO 和人工牵引命令。

## 7.4 仿真时钟与生命周期

### 7.4.1 Tick 配置

Phase 1 推荐：

```text
stepMs = 100
updateFrequency = 10 Hz
mode = REALTIME 或 FAST
```

### 7.4.2 生命周期状态

```text
IDLE
  → LOADED
  → RUNNING
  → PAUSED
  → STOPPED
  → IDLE
```

异常：

```text
ERROR
```

### 7.4.3 调度顺序

Phase 1 调度顺序：

```text
1. EnvironmentService.update()
2. TrackQueryService.prepareSnapshot()
3. TrainControlService.update()
4. PowerService.update()
5. CabControlService.update()
6. VehicleDynamicsService.update()
7. StationService.update()
8. Recorder.record()
9. UiPresenter.update()
```

说明：信号列控先基于上一 tick TrainState 计算安全约束，车辆动力学再执行本 tick 命令。这会产生 1 tick 延迟，但逻辑清晰且便于实现。

先采用一维纵向运动模型：

```text
net_force = traction_force - brake_force - resistance_force - gradient_force
acceleration = net_force / mass
speed_next = max(0, speed + acceleration * dt)
distance = speed * dt + 0.5 * acceleration * dt^2
position_next = TrackNavigator.advance(distance)
```

第一版参数：

```json
{
  "trainId": "T001",
  "massKg": 180000,
  "maxSpeedMps": 22.22,
  "maxTractionLevel": 5,
  "maxBrakeLevel": 5,
  "tractionForcePerLevelN": 20000,
  "brakeForcePerLevelN": 25000,
  "basicResistanceN": 3000,
  "stopSpeedThresholdMps": 0.05
}
```

### 8.1.5 牵引/制动力计算

```text
traction_force = tractionLevel * tractionForcePerLevelN * tractionLimitRatio
brake_force = brakeLevel * brakeForcePerLevelN
if emergencyBrake:
    brake_force = emergencyBrakeForceN
```

约束：

```text
牵引和制动不应同时生效
若 emergencyBrake=true，则 tractionLevel 强制为 0
若 speed_next < 0，则 speed_next = 0
```

### 8.1.6 坡度处理

Phase 1 可提供两种模式：

```text
ignoreGradient=true：不计坡度，只记录坡度字段
ignoreGradient=false：使用已确认单位计算 gradient_force
```

在坡度单位未确认前，默认 `ignoreGradient=true`。

## 8.2 司机台与 ATO 控制模块

### 8.2.1 职责

`CabControlService` 负责将人工手柄、ATO 算法和 ATP 覆盖合成为最终 `ControlCommand`。

### 8.2.2 模式

| 模式 | 说明 |
|---|---|
| MANUAL | 直接采用司机输入 |
| ATO | 根据目标速度/目标距离自动生成指令 |
| ATP_SUPERVISED | 人工驾驶，但 ATP 可覆盖 |
| SH | 调车/降级模式，Phase 1 可只保留状态 |

### 8.2.3 ATO 第一版控制策略

第一版采用规则控制器：

```text
distance = distanceToTargetM
v = speedMps
brake_distance = v^2 / (2 * expectedDeceleration)

if emergencyBrakeRequired:
    emergencyBrake = true
elif distance <= stopTolerance and v <= stopSpeedThreshold:
    traction = 0, brake = holdBrakeLevel
elif distance <= brake_distance + brakeMargin:
    traction = 0, brake = computeBrakeLevel(v, distance)
elif v < targetCruiseSpeed and v < permittedSpeed:
    traction = computeTractionLevel(v, targetCruiseSpeed)
else:
    traction = 0, brake = 0
```

推荐参数：

```json
{
  "targetCruiseSpeedMps": 12.0,
  "expectedDecelerationMps2": 0.8,
  "brakeMarginM": 20.0,
  "stopToleranceM": 1.0,
  "holdBrakeLevel": 1,
  "maxTractionLevel": 4,
  "maxBrakeLevel": 4
}
```

### 8.2.4 ATP 覆盖规则

最终命令合成顺序：

```text
driver/ATO command
  ↓
apply power limit
  ↓
apply speed limit guard
  ↓
apply ATP emergency override
  ↓
publish final ControlCommand
```

若 `SignalState.emergencyBrakeRequired=true`：

```json
{
  "tractionLevel": 0,
  "brakeLevel": 0,
  "emergencyBrake": true,
  "source": "ATP_OVERRIDE"
}
```

## 18. 结论

本设计文档将项目第一阶段收束为一条可落地的主线：

```text
真实线路数据导入
  → 电子地图查询
  → 单列车沿 Seg 拓扑运行
  → ATO 自动控制
  → 基本限速/MA/ATP 防护
  → 到站停车与指标记录
```

Phase 0 和 Phase 1 完成后，系统即可形成一个有真实数据、有运行闭环、有安全约束、有指标输出的可演示原型。Phase 2 再扩展联锁、车站、供电和客流，Phase 3 再接入 AI 优化与智能调度。
